    **1. Create gem tier change detection logic:**

    Add a pure function `detectGemTierChanges` (export from `apps/web/src/lib/gems/constants.ts` or a new `apps/web/src/lib/gems/notifications.ts` file):

    ```typescript
    export interface GemTierChange {
      releaseId: string;
      releaseTitle: string;
      discogsId: number | null;
      oldTier: GemTier;
      newTier: GemTier;
      oldWeight: number;
      newWeight: number;
    }

    export function detectGemTierChanges(
      oldScores: Map<string, { score: number; title: string; discogsId: number | null }>,
      newScores: Map<string, { score: number; title: string; discogsId: number | null }>,
    ): GemTierChange[]
    ```

    Logic: For each entry in newScores, if the entry exists in oldScores, compare `getGemTier(oldScore)` vs `getGemTier(newScore)`. If different, add to changes array. If entry does NOT exist in oldScores (new record), skip (no notification for initial add — prevents first-sync flood per RESEARCH.md Pitfall 2).
