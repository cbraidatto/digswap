### Gem Tier Colors (NEW — phase-specific palette)

Each gem tier has a dedicated color used ONLY for that tier's badge, icon, glow effects, and vault bar. These colors exist outside the 60/30/10 base palette and are used as semantic indicators, similar to how `--tertiary` and `--secondary` are used for scoring elements.

| Gem Tier | Key | Hex | OKLCH Approximation | CSS Custom Property |
|----------|-----|-----|---------------------|---------------------|
| Quartzo | `quartz` | `#9CA3AF` | `oklch(0.72 0.02 260)` | `--gem-quartz` |
| Ametista | `amethyst` | `#8B5CF6` | `oklch(0.55 0.22 290)` | `--gem-amethyst` |
| Esmeralda | `emerald` | `#10B981` | `oklch(0.68 0.16 165)` | `--gem-emerald` |
| Rubi | `ruby` | `#EF4444` | `oklch(0.60 0.22 25)` | `--gem-ruby` |
| Safira | `sapphire` | `#3B82F6` | `oklch(0.58 0.20 260)` | `--gem-sapphire` |
| Diamante | `diamond` | `#F0F9FF` | `oklch(0.97 0.01 240)` | `--gem-diamond` |

**Where gem colors are used (exhaustive):**
- GemBadge pill: border color, text color, icon fill
- GemBadge glow/pulse/shimmer: `box-shadow` and `background` using gem color at various opacities
- GemVault distribution bars: `background-color` for each tier's bar
- GemVault rarest gem highlight: border accent on the rarest tier row
- Notification row icon: gem color on the Material Symbols icon for gem tier change notifications
- Leaderboard row: NOT used (leaderboard continues to use `--primary` for scores)

**Where gem colors are NOT used:**
- Navigation, buttons, form elements, page backgrounds
- Any non-gem UI surface

## Component Inventory

### 1. GemBadge (replaces RarityPill)

**File:** `apps/web/src/components/ui/gem-badge.tsx`
**Replaces:** `apps/web/src/components/ui/rarity-pill.tsx`
**Used in:** 9 consumer files (see RESEARCH.md touch surface)

| Property | Value |
|----------|-------|
| Shape | Pill (rounded-full), matches existing RarityPill exactly |
| Size | `px-2 py-0.5` (8px x 2px padding) |
| Font | `font-mono text-[10px] font-semibold` |
| Layout | `inline-flex items-center gap-1` (icon + tier name + optional weight) |
| Icon size | 12px (Lucide `size={12}`) |
| Border | 1px solid, tier-colored at 25% opacity |
| Background | Tier-colored at 10% opacity |
| Text | Tier color at 100% |

**Props:**
```typescript
interface GemBadgeProps {
  score: number | null | undefined;  // raw rarity score (want/have ratio)
  showScore?: boolean;               // show weight multiplier (default: false)
  className?: string;                // additional Tailwind classes
}
```

**States:**
- `score === null/undefined`: renders nothing (returns null)
- `showScore === true`: appends `x{weight}` at 70% opacity after tier name
- Hover: triggers tier-specific visual effect (see GemEffects below)
- `prefers-reduced-motion`: all effects disabled, static badge only

**Accessibility:**
- Badge includes `aria-label="{TierName} gem (weight {N})"` for screen readers
- Hover effects are decorative only; no information is gated behind hover

Lucide icon mapping per tier:

### 3. GemEffects (CSS animations)

**File:** `apps/web/src/app/globals.css` (appended to existing keyframes section)

#### Quartzo — No effects
Static badge. No hover animation, no glow.

#### Ametista — Subtle purple glow on hover
```css
@keyframes gem-glow-purple {
  0%, 100% { box-shadow: 0 0 4px oklch(0.55 0.22 290 / 0.2); }
  50% { box-shadow: 0 0 10px oklch(0.55 0.22 290 / 0.4); }
}
```
- Trigger: `hover` on GemBadge
- Duration: `1.5s`
- Timing: `ease-in-out`
- Iteration: `infinite` while hovering

#### Esmeralda — Green shimmer on hover
```css
@keyframes gem-shimmer-green {
  0% { background-position: -200% center; }
  100% { background-position: 200% center; }
}
```
- Applied as: `background: linear-gradient(90deg, transparent 40%, oklch(0.68 0.16 165 / 0.15) 50%, transparent 60%); background-size: 200% 100%;`
- Trigger: `hover`
- Duration: `2s`
- Timing: `linear`
- Iteration: `infinite` while hovering

#### Rubi — Warm pulse on hover
```css
@keyframes gem-pulse-warm {
  0%, 100% { box-shadow: 0 0 4px oklch(0.60 0.22 25 / 0.3); }
  50% { box-shadow: 0 0 14px oklch(0.60 0.22 25 / 0.6); }
}
```
- Trigger: `hover`
- Duration: `1.2s`
- Timing: `ease-in-out`
- Iteration: `infinite` while hovering

#### Safira — Blue glow + sparkle particles on hover
```css
@keyframes gem-glow-blue {
  0%, 100% { box-shadow: 0 0 6px oklch(0.58 0.20 260 / 0.3); }
  50% { box-shadow: 0 0 16px oklch(0.58 0.20 260 / 0.5); }
}

@keyframes gem-sparkle {
  0%, 100% { opacity: 0; transform: scale(0) rotate(0deg); }
  50% { opacity: 1; transform: scale(1) rotate(180deg); }
}
```
- Blue glow: same as Ametista but stronger radius (6px to 16px)
- Sparkle: 2 CSS pseudo-elements (`::before`, `::after`) positioned at top-right and bottom-left
- Sparkle pseudo-element: `content: ""; width: 3px; height: 3px; border-radius: 50%; background: oklch(0.58 0.20 260);`
- `::before` animation-delay: `0s`, `::after` animation-delay: `0.6s`
- Trigger: `hover`
- Duration: `1.2s` (sparkle), `1.5s` (glow)

#### Diamante — Prismatic refraction (ALWAYS ACTIVE)
```css
@keyframes gem-prismatic {
  0% { filter: hue-rotate(0deg) brightness(1.1); }
  25% { filter: hue-rotate(90deg) brightness(1.2); }
  50% { filter: hue-rotate(180deg) brightness(1.3); }
  75% { filter: hue-rotate(270deg) brightness(1.2); }
  100% { filter: hue-rotate(360deg) brightness(1.1); }
}

@keyframes gem-rainbow-edge {
  0% { border-color: oklch(0.70 0.20 0); }
  16% { border-color: oklch(0.70 0.20 60); }
  33% { border-color: oklch(0.70 0.20 120); }
  50% { border-color: oklch(0.70 0.20 180); }
  66% { border-color: oklch(0.70 0.20 240); }
  83% { border-color: oklch(0.70 0.20 300); }
  100% { border-color: oklch(0.70 0.20 360); }
}
```
- Prismatic: `animation: gem-prismatic 4s linear infinite` — ALWAYS active, not just on hover
- Rainbow edge: `animation: gem-rainbow-edge 3s linear infinite` — border cycles through hue
- Sparkle: same as Safira but always active, 3 pseudo-elements instead of 2
- Combined: the Diamante badge shimmer + shifts border color + sparkle particles all at once
- This is the "this record is truly special" visual treatment

#### Reduced Motion
All gem animations MUST be wrapped in the existing `@media (prefers-reduced-motion: reduce)` block in globals.css. Under reduced motion:
- All keyframe animations disabled
- Diamante gets a static `border-color: oklch(0.70 0.20 290)` (purple, suggesting prismatic without motion)
- All glow effects reduce to a static subtle `box-shadow` at rest state values

### 4. GemVault (profile section)

**File:** `apps/web/src/app/(protected)/(profile)/perfil/_components/gem-vault.tsx`
**Placement:** Replaces the current "Rarity" label in RankCard + adds a new dedicated section in AboutTab
**Layout:** Horizontal stacked bar chart (CSS flexbox, no charting library)

| Property | Value |
|----------|-------|
| Container | `bg-surface-container-low rounded-xl p-5 border border-outline-variant/5` (matches existing AboutTab section pattern) |
| Section title | "Gem Vault" — `font-mono text-[10px] text-on-surface-variant uppercase tracking-widest` with Material Symbols `diamond` icon at 14px |
| Total score | `text-2xl font-bold font-heading text-primary` with CountUp animation |
| Score label | "Gem Score" — `font-mono text-[9px] text-on-surface-variant/50 uppercase tracking-widest` |

**Distribution Bar:**
- 6 segments in a single horizontal bar, proportional to count per tier
- Total bar height: 28px, border-radius: 6px (rounded-md)
- Each segment colored with its tier's gem color
- Minimum segment width: 4px (so even 1 record of a tier is visible)
- Segments ordered left-to-right: Quartzo, Ametista, Esmeralda, Rubi, Safira, Diamante
- Below the bar: 6 columns showing tier icon + count

**Tier Breakdown Grid:**
- Grid: `grid grid-cols-3 md:grid-cols-6 gap-3`
- Each cell: gem icon (16px) + tier name (9px mono uppercase) + count (14px mono semibold)
- Rarest gem tier (highest-weight tier with count > 0) gets a left border in its gem color (`border-l-2 border-[gemColor]`)
- If user has 0 records in a tier, show `0` with 30% opacity

**Props:**
```typescript
interface GemVaultProps {
  distribution: Record<GemTier, number>;  // count per tier
  totalGemScore: number;                  // pre-computed weighted sum
  totalRecords: number;                   // for percentage calculations
}
```

## Interaction Contracts

### Hover Effects (GemBadge)

| Tier | Trigger | Effect | Duration | Easing |
|------|---------|--------|----------|--------|
| Quartzo | hover | None | — | — |
| Ametista | hover | Purple glow box-shadow pulses | 1.5s | ease-in-out |
| Esmeralda | hover | Green shimmer slides across background | 2s | linear |
| Rubi | hover | Red-orange box-shadow pulse, stronger intensity | 1.2s | ease-in-out |
| Safira | hover | Blue glow pulse + 2 sparkle dots rotate in/out | 1.2s / 1.5s | ease-in-out |
| Diamante | always | Prismatic hue-rotate + rainbow border + 3 sparkle dots | 4s / 3s / 1.2s | linear |

### GemVault Interactions

- Distribution bar segments: `cursor-default` (not clickable)
- Total gem score: animated entrance with CountUp component (matches existing pattern in AboutTab)
- Tier breakdown cells: `cursor-default` (informational only)
- No drill-down interaction in v1 (clicking a tier to see those records is deferred)

### Notification Interactions

- Gem tier change notification row: clickable, navigates to `/release/{discogsId}`
- Read/unread state: follows existing notification system (unread has `border-l-2 border-primary`)
- No special animation on the notification itself

## CSS Custom Properties (to add to globals.css)

Add these inside the `:root, [data-theme="ghost"]` block and replicate for each theme:

```css
/* Gem tier colors */
--gem-quartz: #9CA3AF;
--gem-amethyst: #8B5CF6;
--gem-emerald: #10B981;
--gem-ruby: #EF4444;
--gem-sapphire: #3B82F6;
--gem-diamond: #F0F9FF;
```

These are NOT theme-dependent (gem colors are fixed across all themes for brand consistency). Declare them once in `:root` only.

Register in Tailwind `@theme inline` block:
```css
--color-gem-quartz: var(--gem-quartz);
--color-gem-amethyst: var(--gem-amethyst);
--color-gem-emerald: var(--gem-emerald);
--color-gem-ruby: var(--gem-ruby);
--color-gem-sapphire: var(--gem-sapphire);
--color-gem-diamond: var(--gem-diamond);
```

This enables `text-gem-ruby`, `bg-gem-sapphire/10`, `border-gem-diamond/25` etc. in Tailwind classes, replacing raw hex values for maintainability.
