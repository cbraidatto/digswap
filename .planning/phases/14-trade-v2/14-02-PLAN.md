# Plan 14-02: Proposal Form Redesign

## Goal
Redesign the trade proposal form to capture quality metadata and offering release instead of file selection, implementing the "metadata only at proposal time" principle from D-02.

## Context
Implements D-01 (asymmetric proposal structure), D-02 (file is ephemeral — no file in proposal), D-05 (quality metadata: format, declared quality, condition notes). The form captures what the proposer is offering (a release from their collection) and what they want (the target release), plus quality specs. File selection moves to the lobby (Plan 14-03/14-04).

## Tasks

### Task 1: Update createTrade server action

**Files:**
- `src/actions/trades.ts` -- rewrite createTrade signature and logic

**Steps:**

1. Change the `createTrade` function signature from:
   ```typescript
   export async function createTrade(formData: {
     providerId: string;
     releaseId?: string;
     fileName: string;
     fileFormat: string;
     declaredBitrate: string;
     fileSizeBytes: number;
     message?: string;
   })
   ```
   To:
   ```typescript
   export async function createTrade(formData: {
     providerId: string;
     releaseId?: string;
     offeringReleaseId: string;
     declaredQuality: string;
     conditionNotes: string;
     message?: string;
   })
   ```

2. Add validation before the DB insert:
   ```typescript
   if (!formData.offeringReleaseId) {
     return { error: "You must select a release you're offering" };
   }
   if (!formData.conditionNotes || formData.conditionNotes.trim().length < 10) {
     return { error: "Condition notes must be at least 10 characters" };
   }
   if (!formData.declaredQuality || formData.declaredQuality.trim().length === 0) {
     return { error: "Declared quality is required" };
   }
   ```

3. Update the insert object to use new fields and set status to `'lobby'` instead of `'pending'`:
   ```typescript
   const { data: trade, error: insertError } = await admin
     .from("trade_requests")
     .insert({
       requester_id: user.id,
       provider_id: formData.providerId,
       release_id: formData.releaseId || null,
       offering_release_id: formData.offeringReleaseId,
       declared_quality: formData.declaredQuality.trim(),
       condition_notes: formData.conditionNotes.trim(),
       status: "lobby",
       message: formData.message?.trim() || null,
       expires_at: new Date(Date.now() + TRADE_EXPIRY_HOURS * 60 * 60 * 1000).toISOString(),
       terms_accepted_at: new Date().toISOString(), // Proposer implicitly accepts their own terms on creation
     })
     .select("id")
     .single();
   ```
   Remove `file_name`, `file_format`, `declared_bitrate`, `file_size_bytes` from the insert — these are set in the lobby, not at proposal time.

### Task 3: Update /trades/new/page.tsx to load user collection

**Files:**
- `src/app/(protected)/trades/new/page.tsx` -- fetch user's collection for the offering picker

**Steps:**

1. After the existing `counterparty` and `releaseInfo` fetches, add a collection query:
   ```typescript
   // Fetch user's collection for offering release picker
   const { data: collectionItems } = await admin
     .from("user_collections")
     .select("release_id, releases(id, title, artist, thumbnail_url)")
     .eq("user_id", user.id)
     .order("created_at", { ascending: false })
     .limit(500);

   const userCollection = (collectionItems ?? [])
     .filter((item: any) => item.releases)
     .map((item: any) => ({
       releaseId: item.releases.id,
       title: item.releases.title ?? "Unknown",
       artist: item.releases.artist ?? "Unknown",
       thumbnailUrl: item.releases.thumbnail_url ?? null,
     }));
   ```

2. Pass `userCollection` to the TradeForm:
   ```tsx
   <TradeForm
     toUserId={toUserId ?? null}
     releaseId={releaseId ?? null}
     counterpartyUsername={counterparty?.username ?? null}
     counterpartyAvatar={counterparty?.avatarUrl ?? null}
     releaseTitle={releaseInfo?.title ?? null}
     releaseArtist={releaseInfo?.artist ?? null}
     userCollection={userCollection}
   />
   ```
