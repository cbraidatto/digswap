  <behavior>
    createProposalAction:
    - Free user: 1 offer item + 1 want item max. Returns error "Free tier allows 1 item per side. Upgrade to Premium for multi-item trades." if exceeded
    - Premium user: up to 3 offer + 3 want
    - Each item MUST have declaredQuality. Returns error "All items must have a declared quality." if any missing
    - Creates tradeRequests row (if no existing trade between pair for this release combo) OR finds existing pending trade, then inserts tradeProposals + tradeProposalItems
    - Backward compat: also writes releaseId + offeringReleaseId to trade_requests (first offer/want item's releaseId)
    - Returns { proposalId, tradeId } on success

    createCounterproposalAction:
    - Turn enforcement: caller must be the OTHER participant (not the last proposer). Returns error "It is not your turn to propose." if violation
    - Max rounds: returns error "Maximum of 10 counterproposal rounds reached." when currentMaxSequence >= 10
    - Supersedes previous pending proposal (UPDATE tradeProposals SET status = 'superseded' WHERE tradeId = X AND status = 'pending')
    - Inserts new tradeProposals with sequenceNumber = currentMax + 1
    - Inserts tradeProposalItems for new proposal
    - Same tier + quality rules as createProposalAction
    - Inserts counterproposal_received notification for the OTHER participant via admin client
    - Returns { proposalId } on success

    acceptProposalAction:
    - Caller must be the OTHER participant (not the proposer). Returns error "Only the recipient can accept a proposal."
    - Proposal must be status = 'pending'. Returns error if not
    - Updates tradeProposals SET status = 'accepted'
    - Updates tradeRequests SET status = 'accepted' for the parent trade
    - Returns { success: true }

    declineProposalAction:
    - Caller must be OTHER participant
    - Updates tradeProposals SET status = 'rejected'
    - Updates tradeRequests SET status = 'declined'
    - Returns { success: true }

    Tests cover:
    - createProposalAction: happy path 1:1, free tier limit error (2 items), missing quality error, unauthenticated

    - acceptProposalAction: happy path, wrong participant error, non-pending proposal error
    - declineProposalAction: happy path, wrong participant error
