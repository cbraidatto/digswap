**tests/unit/release/youtube-search.test.ts:**
Tests for searchYouTubeForRelease server action.

Mock: src/lib/supabase/server (createClient for auth), src/lib/rate-limit (apiRateLimit.limit), src/lib/db (Drizzle db for release lookup), src/lib/supabase/admin (createAdminClient for cache write), global fetch for YouTube API.

Test cases:
1. `returns cached videoId without calling YouTube API` -- mock release with youtubeVideoId already set. Assert returned videoId matches, assert fetch NOT called.
2. `calls YouTube API and caches result on cache miss` -- mock release with null youtubeVideoId. Mock fetch returning YouTube search response with videoId "dQw4w9WgXcQ". Assert returned videoId, assert admin client update called with youtube_video_id.
3. `returns null when YouTube API returns no results` -- mock fetch returning { items: [] }. Assert { videoId: null } returned.
4. `returns null when YouTube API returns error (quota exceeded)` -- mock fetch returning response with ok=false. Assert { videoId: null } returned, no crash.
5. `returns null when YOUTUBE_API_KEY is not set` -- set process.env.YOUTUBE_API_KEY to undefined. Assert { videoId: null } returned.
6. `returns error when user is not authenticated` -- mock getUser returning null user. Assert { videoId: null, error: "Authentication required to trigger search" }.
7. `returns error when rate limited` -- mock apiRateLimit.limit returning { success: false }. Assert { videoId: null, error: "Rate limited" }.
