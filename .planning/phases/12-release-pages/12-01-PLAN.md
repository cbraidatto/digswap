  <name>Task 1: Release query functions + YouTube server action</name>
  <files>src/lib/release/queries.ts, src/actions/release.ts</files>
  <action>
Create src/lib/release/queries.ts with three functions and one interface, all using Drizzle db client (bypasses RLS for public page access per existing pattern):

1. `getReleaseByDiscogsId(discogsId: number)` -- selects all columns from releases table where releases.discogsId = param, returns release row or null. Import `sql` from drizzle-orm for the count query.

2. `ReleaseOwner` interface: `{ userId: string; username: string | null; avatarUrl: string | null; displayName: string | null; conditionGrade: string | null }`

3. `getOwnersByReleaseId(releaseId: string, limit = 12)` -- joins collection_items with profiles on userId, filters by releaseId, returns ReleaseOwner[]. The limit=12 default matches the locked decision "top 12 by default".

4. `getOwnerCountByReleaseId(releaseId: string)` -- returns count of collection_items for the release as number. Use `sql<number>\`count(*)::int\`` pattern (same as existing queries).

Create src/actions/release.ts with the YouTube lazy-cache server action:

1. `searchYouTubeForRelease(releaseInternalId: string)` -- "use server" directive at top.
   - Authenticate user via createClient().auth.getUser() -- only logged-in users can trigger search (rate limiting needs user ID).
   - Rate limit with apiRateLimit.limit(user.id) from src/lib/rate-limit.ts.
   - Fetch release from DB via Drizzle db client. If release.youtubeVideoId already set, return it (cache hit).
   - If YOUTUBE_API_KEY env var is not set, return { videoId: null } gracefully (no crash).
   - Call YouTube Data API v3 search.list: `https://www.googleapis.com/youtube/v3/search` with params: part=snippet, q="{artist} {title}", type=video, videoCategoryId=10 (Music), maxResults=1, key=YOUTUBE_API_KEY.
   - If response not ok (quota exceeded, API error), return { videoId: null } -- fail gracefully per locked decision.
   - Extract videoId from data.items[0]?.id?.videoId.
   - If videoId found, write to DB via createAdminClient() from src/lib/supabase/admin.ts: update releases set youtube_video_id = videoId where id = release.id. Use Supabase admin client (not Drizzle) for the write -- consistent with existing mutation pattern.
   - Return { videoId }.
   - Return type: `Promise<{ videoId: string | null; error?: string }>`

Follow the exact code structure from 12-RESEARCH.md "YouTube Data API v3 Search" code example. Do NOT store null results -- if no video found, leave the column null so future searches can retry.
