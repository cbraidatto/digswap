## Interaction Specifications

### Trade Inbox Tab Switching

- URL updates to `?tab=pending|active|completed` on tab selection
- Default tab: PENDING
- Tab content renders immediately (client-side state)
- Tab counts shown as badge: `PENDING (3)`

### Trade Request Flow

1. User clicks "Request Audio" / "Request Trade" entry point
2. If `trades_tos_accepted_at` is null: ToS modal appears (blocking)
3. If `tradesThisMonth >= 5 && plan === 'free'`: Quota reached message shown instead of form
4. Otherwise: Navigate to `/trades/new?to=[userId]&release=[releaseId]`
5. User selects audio file, sets metadata, chooses expiry, clicks SEND_TRADE_REQUEST
6. Server action creates `tradeRequests` row with status `pending`
7. Notification sent to recipient (in-app + email per NOTF-02)
8. User redirected to `/trades` inbox

### Trade Lobby Flow

1. Both parties navigate to `/trades/[id]` (from notification or inbox)
2. Supabase Realtime subscription on `trade_requests` row for status changes
3. First arrival: WAITING state with blinking cursor
4. Second arrival: Status transitions to CONNECTING, PeerJS initializes
5. PeerJS connection established: Status transitions to TRANSFERRING
6. File chunks sent/received with progress updates
7. Transfer complete: Status transitions to COMPLETE, redirect to `/trades/[id]/complete`
8. If connection drops: Status transitions to FAILED with retry option

### File Upload (Trade Form)

- Drag-and-drop zone accepts audio MIME types: `audio/flac`, `audio/wav`, `audio/mp3`, `audio/mpeg`, `audio/ogg`, `audio/aac`
- Click-to-browse fallback with hidden file input
- Max file size: Display warning at 500MB+ (no hard block -- WebRTC handles large files via chunking)
- After file selection: Display file name, detected format, file size
- Drop zone visual states:
  - Default: `border-2 border-dashed border-outline-variant/30`
  - Drag over: `border-2 border-dashed border-primary/50 bg-primary/5`
  - File selected: `border-2 border-solid border-primary/30 bg-surface-container-lowest`

### Spectrogram Interaction

- Renders automatically on trade review page (1 free analysis per trade)
- Canvas uses `requestAnimationFrame` for smooth rendering
- Color palette: gradient from `--surface-container-lowest` (silence) to `--primary` (peak amplitude)
- Premium gate: After first render, free users see blur overlay with lock icon
- Premium users can click "Re-analyze" to regenerate with different FFT settings

### Star Rating (Trade Complete)

- 5 stars in a row, each 40x40px (`w-10 h-10`)
- Hover: stars up to hovered position fill with `text-secondary`
- Click: locks selection, filled stars stay `text-secondary`
- Unselected stars: `text-on-surface-variant`
- Rating required before SUBMIT_REVIEW (button disabled without selection)
- Comment is optional

### Trade Decline/Cancel Confirmation

- Decline (recipient rejecting a request): AlertDialog with confirmation
- Cancel (requester withdrawing): AlertDialog with confirmation
- Both use existing `alert-dialog` shadcn component

### Empty States

- Trade inbox (no trades): Show empty state (see Copywriting Contract)
- Trade inbox (tab empty): Show tab-specific empty state
- Lobby (waiting): Blinking cursor animation (existing `.blink` CSS class)

### Loading States

- Trade inbox: 3 skeleton rows (56px height each, matching TradeRow height)
- Lobby: Skeleton card with pulse animation
- Spectrogram: Aspect-video skeleton with shimmer
- Spec check table: 5 skeleton rows matching existing grid layout

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab role semantics | PENDING/ACTIVE/COMPLETED tabs use `role="tablist"` and `role="tab"` with `aria-selected` |
| Trade row as list | Wrap rows in `<ul>` with `role="list"`, each row is `<li>` with `role="listitem"` |
| Lobby state announcements | Use `aria-live="polite"` on lobby state text for screen reader updates on state transitions |
| Transfer progress | Progress bar uses `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"` |
| File upload | Drop zone has `role="button"` with `aria-label="Select audio file to upload"`, hidden file input linked via label |
| Star rating | Rating group uses `role="radiogroup"` with `aria-label="Audio quality rating"`, each star is `role="radio"` with `aria-checked` |
| ToS modal | Modal uses `role="alertdialog"` with `aria-modal="true"`, focus trapped within modal |
| Decline/cancel dialogs | AlertDialog components have proper focus management (handled by shadcn/Radix) |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Ghost Protocol palette verified in Phase 1 |
| Reduced motion | All animations (blink cursor, transfer progress, spectrogram) respect `prefers-reduced-motion` (existing global rule) |
| Keyboard navigation | All interactive elements reachable via Tab. Star rating navigable via arrow keys. Drop zone activable via Enter/Space |
