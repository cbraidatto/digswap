Hook signature:
```typescript
export interface PeerState {
  status: "idle" | "waiting" | "connecting" | "transferring" | "complete" | "failed";
  progress: number; // 0-100
  bytesTransferred: number;
  totalBytes: number;
  speed: number; // bytes/sec
  eta: number; // seconds remaining
  error: string | null;
  receivedFile: Blob | null;
  receivedFileName: string | null;
}

export function usePeerConnection(
  tradeId: string,
  userId: string,
  role: "sender" | "receiver",
  iceServers: RTCIceServer[],
  file?: File,
): { state: PeerState; sendFile: () => Promise<void>; retry: () => void; cleanup: () => void }
```

Implementation details:
- **Peer ID format:** `digswap-${tradeId}-${userId.substring(0, 8)}` (deterministic, per Pitfall 3 from RESEARCH.md)
- **Counterpart Peer ID:** `digswap-${tradeId}-${counterpartId.substring(0, 8)}` — but since the hook doesn't know counterpart ID, use role-based pattern: sender creates peer and waits for connection via `peer.on("connection")`; receiver connects to sender's peer ID
- **Alternative approach (simpler):** Both parties register with their own peer ID. The sender calls `peer.connect(receiverPeerId)`. The receiver peer ID is known from the trade request data (providerId or requesterId).
- Initialize Peer with: `new Peer(peerId, { config: { iceServers }, debug: 0 })`
- On `peer.on("open")`: set status to "waiting"
- On `peer.on("connection")` (incoming): set status to "connecting", store DataConnection reference
- To connect outbound: `peer.connect(counterpartPeerId)`
- On DataConnection `open`: set status to "transferring" (if sender and file provided, begin sending)
- **Sender flow:** Call `sliceFileIntoChunks(file)`, iterate chunks, send each via `conn.send()`. Implement flow control: check `(conn as any)._dc.bufferedAmount > 256 * 1024` and wait for `bufferedamountlow` event. Update progress via `calculateTransferStats()`.
- **Receiver flow:** Collect chunks into array, update progress. On "done" message, call `reassembleChunks()`, set receivedFile and receivedFileName, set status "complete".
- On `peer.on("error")`: set status "failed", set error message
- On `conn.on("close")`: if not complete, set status "failed"
- **Retry:** destroy current peer, reinitialize
- **Cleanup on unmount:** `peer.destroy()`, close connections (useEffect cleanup)
- Respect `prefers-reduced-motion` — no visual concern in hook, but transfer progress updates throttled to every 100ms to avoid excessive re-renders
