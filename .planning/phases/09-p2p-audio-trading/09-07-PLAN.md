    **Also affects skipReview:** The `skipReview` function (lines 456-509) has the same two omissions — it awards badges but never increments `trades_this_month` or `contribution_score` for either party.

    **Fix 1 — Replace tradesThisMonth RPC block (lines 397-417) in completeTrade with read-increment-write:**
    Replace the entire block (the for-loop with `admin.rpc("increment_field")` and its fallback) with:
    ```typescript
    // Increment tradesThisMonth for both parties (read-increment-write)
    for (const oderId of [trade.requester_id, trade.provider_id]) {
      const { data: sub } = await admin
        .from("subscriptions")
        .select("trades_this_month")
        .eq("user_id", oderId)
        .maybeSingle();

      await admin
        .from("subscriptions")
        .update({
          trades_this_month: (sub?.trades_this_month ?? 0) + 1,
          updated_at: new Date().toISOString(),
        })
        .eq("user_id", oderId);
    }
    ```

    Replace the entire block (the for-loop that only writes `updated_at` to subscriptions) with:
    ```typescript
    // Increment contribution scores (+15 per trade_completed) in user_rankings
    for (const oderId of [trade.requester_id, trade.provider_id]) {
      const { data: ranking } = await admin
        .from("user_rankings")
    ```
