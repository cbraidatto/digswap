## Correction note (Plan 04 retroactively fixed Plan 03)

Initial population of `trade_preview_publishable_key` used the modern publishable key
format (`sb_publishable_*`). During Plan 04 smoke testing, the cleanup-trade-previews
Edge Function returned 401 `UNAUTHORIZED_INVALID_JWT_FORMAT` — the gateway with
verify_jwt=true requires JWT format, and `sb_publishable_*` is an API key, not a JWT.

Resolution: deleted the modern publishable entry, re-inserted the legacy anon JWT
(format-compatible with verify_jwt). The function still uses service_role internally
for actual work; the bearer is just to pass the gateway.

## Why legacy JWT, not modern publishable

- Edge Function gateway with `verify_jwt=true` requires JWT format
- Legacy anon `eyJhbGciOiJIUzI1NiI...` IS a JWT (passes gateway)
- Modern `sb_publishable_*` is an API key (fails gateway)
- The cleanup-trade-previews function uses Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")
  for its actual database/storage work — the bearer is just gateway-pass-through
- This decision applies to every pg_cron job that invokes an Edge Function via
  pg_net + Vault-stored bearer; future similar wiring should default to the legacy
  anon JWT for the bearer
