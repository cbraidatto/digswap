**Create VisibilitySelector** at `apps/web/src/components/ui/visibility-selector.tsx`:

A compact "use client" dropdown component for setting item visibility. Design:
```typescript
interface VisibilitySelectorProps {
  itemId: string;
  currentVisibility: string;
  compact?: boolean; // true = icon-only, false = icon+label
}
```

The component renders a button that cycles through states on click:
- `not_trading` (default): dim swap_horiz icon, no badge. Label: "Not Trading"
- `tradeable`: highlighted primary-color swap_horiz icon. Label: "Trading"
- `private`: lock icon in muted color. Label: "Private"

On click: cycle `not_trading -> tradeable -> private -> not_trading`. Call `setVisibility(itemId, nextState)` via server action. Use `useTransition` for pending state. Toast on success/error. `router.refresh()` after.

Use Material Symbols icons: `swap_horiz` for not_trading/tradeable, `lock` for private. Colors match existing card styling (primary for tradeable, on-surface-variant/25 for not_trading, on-surface-variant/40 for private).
