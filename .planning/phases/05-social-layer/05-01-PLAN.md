**src/actions/social.ts** ŌĆö Create with `"use server"` directive. Follow the same auth pattern as `src/actions/collection.ts`: `createClient()` for auth, then use Drizzle `db` for queries (not Supabase admin client, since RLS policies already allow users to insert/delete their own follows and activity).

Server actions:
1. `followUser(targetUserId: string): Promise<{ success?: boolean; error?: string }>` ŌĆö Auth check via `getUser()`. Guard: `if (targetUserId === user.id) return { error: "Cannot follow yourself" }`. Insert into `follows` using `db.insert(follows).values({ followerId: user.id, followingId: targetUserId })`. Wrap in try/catch for unique constraint violation (already following). Call `logActivity(user.id, "followed_user", "user", targetUserId, { username: targetUsername })` ŌĆö fetch target username from profiles for metadata. Per D-11, D-25, D-26.

2. `unfollowUser(targetUserId: string): Promise<{ success?: boolean; error?: string }>` ŌĆö Auth check. Delete from follows: `db.delete(follows).where(and(eq(follows.followerId, user.id), eq(follows.followingId, targetUserId)))`. Per D-12, D-25.

3. `logActivity(userId: string, actionType: string, targetType: string | null, targetId: string | null, metadata: Record<string, unknown> | null): Promise<void>` ŌĆö Insert into `activityFeed` using `db.insert(activityFeed).values(...)`. This is an internal helper, exported but called by other server actions. Per D-25, D-26.

4. `loadMoreFeed(cursor: string | null, mode: "personal" | "global"): Promise<FeedItem[]>` ŌĆö Auth check. Define `FeedItem` type:
   ```typescript
   export interface FeedItem {
     id: string;
     userId: string;
     actionType: string;
     targetType: string | null;
     targetId: string | null;
     metadata: Record<string, unknown> | null;
     createdAt: string; // ISO string for cursor
     // Joined from profiles:
     username: string | null;
     displayName: string | null;
     avatarUrl: string | null;
     // Joined from releases (for added_record events):
     releaseTitle: string | null;
     releaseArtist: string | null;
     releaseGenre: string[] | null;
     releaseLabel: string | null;
     releaseCoverUrl: string | null;
     releaseRarityScore: number | null;
   }
   ```
   For `mode === "personal"`: call `getPersonalFeed(user.id, cursor)` from queries.ts.
   For `mode === "global"`: call `getGlobalFeed(cursor)` from queries.ts.
   Return the result array. Per D-01, D-02, D-27, D-28.

5. `searchUsers(query: string): Promise<SearchResult[]>` ŌĆö Auth check. If `query.trim().length < 2` return []. Define `SearchResult`:
   ```typescript
   export interface SearchResult {
     id: string;

     recordCount: number;
   ```

   Query: `db.select(...).from(profiles).where(ilike(profiles.username, '%' + sanitizedQuery + '%')).limit(20)`. Then for each result, get record count from collectionItems and follower count from follows. Also check if current user is following each result. Per D-15, D-16.

**src/lib/social/queries.ts** ŌĆö Pure query functions (no "use server"). All use the Drizzle `db` client.

1. `getGlobalFeed(cursor: string | null, limit = 20): Promise<FeedItem[]>` ŌĆö Select from `activityFeed` LEFT JOIN `profiles` (on userId) LEFT JOIN `releases` (on targetId where actionType='added_record'). WHERE: `activityFeed.actionType IN ('added_record', 'followed_user')`. If cursor: `AND activityFeed.createdAt < cursorDate`. ORDER BY: `COALESCE(releases.rarityScore, -1) DESC, activityFeed.createdAt DESC`. LIMIT: `limit`. Per D-27.

2. `getPersonalFeed(currentUserId: string, cursor: string | null, limit = 20): Promise<FeedItem[]>` ŌĆö Same joins. WHERE: `activityFeed.userId IN (SELECT followingId FROM follows WHERE followerId = currentUserId)` AND actionType filter AND cursor condition. ORDER BY `activityFeed.createdAt DESC`. Per D-28.

3. `getFollowCounts(userId: string): Promise<{ followingCount: number; followerCount: number }>` ŌĆö Two count queries on follows table. Per D-13, D-14.

4. `getFollowers(userId: string, limit = 20, offset = 0): Promise<FollowUser[]>` ŌĆö Select from follows JOIN profiles WHERE followingId = userId. Return `{ id, username, displayName, avatarUrl }`. Per D-13.

5. `getFollowing(userId: string, limit = 20, offset = 0): Promise<FollowUser[]>` ŌĆö Select from follows JOIN profiles WHERE followerId = userId. Per D-13.

6. `checkIsFollowing(currentUserId: string, targetUserId: string): Promise<boolean>` ŌĆö Single select checking if follow row exists.

7. `getProgressBarState(userId: string): Promise<{ discogsConnected: boolean; followingCount: number }>` ŌĆö Query `profiles.discogsConnected` and count follows where `followerId = userId`. Per D-22, D-24.

Export the `FollowUser` type: `{ id: string; username: string | null; displayName: string | null; avatarUrl: string | null }`.
