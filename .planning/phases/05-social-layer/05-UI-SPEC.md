> Visual and interaction contract for the Social Layer phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (base-nova style) |
| Preset | base-nova (components.json) |
| Component library | @base-ui/react (shadcn base-nova primitives) |

| Component | Phase 5 Usage |
|-----------|---------------|
| Avatar + AvatarImage + AvatarFallback | Profile headers (feed cards, public profile, search results) |

None required. All Phase 5 UI is built from existing installed components + custom Ghost Protocol card components.

## Spacing Scale

Declared values (multiples of 4, inherited from existing codebase):

| Token | Value | Phase 5 Usage |
|-------|-------|---------------|
| xs | 4px | Icon-to-text gaps inside feed cards, badge internal padding |
| sm | 8px | Feed card header element spacing, compact follow-event line padding |
| md | 16px | Default padding inside feed cards, search results list gap, comparison column padding |
| lg | 24px | Section padding in profile headers, progress banner internal padding |
| xl | 32px | Gap between feed cards in the list, comparison columns gap on desktop |
| 2xl | 48px | Feed page top padding, profile header margin-bottom |
| 3xl | 64px | Not used in Phase 5 |

Exceptions: 44px minimum touch target for Follow/Unfollow button (accessibility, matches mobile tap target standard). Achieved via `h-11` (44px) on mobile, `h-8` (32px) on desktop with `md:h-8`.

| Role | CSS Variable | Hex | Usage |
|------|-------------|-----|-------|

### Accent (`#6fdd78` primary green) Reserved For

1. Follow button default state (outlined border + text)
2. Feed card rarity accent strip -- Common tier only
3. Progress banner completed step checkmarks
4. `ARCHIVE_FEED` page title color (existing)
5. Active feed mode indicator
6. Username links on hover
7. Comparison "In common" column count badge

### Secondary Accent (`#aac7ff` secondary blue)

1. Feed card rarity accent strip -- Rare tier
2. Comparison "Unique to you" column header accent
3. Follow count display color on profiles

### Tertiary Accent (`#ffb689` tertiary orange)

1. Feed card rarity accent strip -- Ultra Rare tier
2. Comparison "Unique to them" column header accent
3. Rarity score badge on Ultra Rare feed cards

### Rarity Tier Color Mapping (for feed card accent strips)

| Tier | Score Range | Accent Strip Color | CSS Variable |
|------|------------|-------------------|-------------|
| Common | < 0.5 | Green | `--primary` (`#6fdd78`) |
| Rare | 0.5 - 1.99 | Blue | `--secondary` (`#aac7ff`) |
| Ultra Rare | >= 2.0 | Orange | `--tertiary` (`#ffb689`) |

Source: D-04 from CONTEXT.md, mapped to existing `getRarityTier()` in `src/lib/collection/rarity.ts`.

| Comparison load failed | Query error | `could not load comparison data. try again later.` |
| Search failed | Query error | `search failed. try again.` (inline text below search input) |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Unfollow | No confirmation dialog. Optimistic toggle with visual state change (green -> red on hover). Instant revert on error. Per D-12, this is a lightweight action that doesn't warrant a dialog. |

## Interaction Contracts

### Infinite Scroll (Feed)

1. Page loads with initial 20 items (server-rendered)
2. User scrolls down, sentinel div enters viewport (`useInView` with `threshold: 0`)
3. `startTransition` triggers server action `loadMoreFeed(cursor, mode)`
4. 3 skeleton cards appear below existing items
5. New items append below existing items
6. New cursor is set to `createdAt` of last returned item
7. If 0 items returned, `hasMore` set to false, `[END_OF_FEED]` shown
8. Deduplication: client-side accumulator uses `id` as key (Map or Set)

### Feed Mode Toggle

1. Default mode: `global` (if user follows 0 people) or `personal` (if following >= 1)
2. Toggle resets accumulated items and cursor
3. `startTransition` fetches first page of new mode
4. Subtitle text updates per mode

### Follow/Unfollow (Optimistic)

1. User clicks Follow on public profile
2. `useOptimistic` immediately flips button to `FOLLOWING` state + increments follower count
3. Server action `followUser` executes
4. On success: optimistic state becomes real state
5. On error: optimistic state reverts, toast error shown via Sonner
6. Same pattern for Unfollow (decrements count)

### Username Search (Explorar)

1. User types in search input
2. After 300ms debounce, if >= 2 characters, server action `searchUsers(query)` fires
3. Results render below search input
4. Each result has inline Follow button
5. Clearing input clears results

### Follower/Following List (Expand)

1. User clicks follower or following count text
2. List expands inline below counts (CSS transition, max-height animation)
3. Clicking again collapses the list
4. List is paginated (load first 20, "show more" link if > 20)
