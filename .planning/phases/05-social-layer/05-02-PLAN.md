- Container: `bg-surface-container-low p-1 rounded-lg flex items-center gap-1 mb-6`
- Active button: `bg-primary text-on-primary text-[10px] font-bold rounded px-3 py-1.5`
- Inactive button: `text-on-surface-variant hover:text-on-surface text-[10px] font-bold px-3 py-1.5`
- Use `role="tablist"` on container, `role="tab"` + `aria-selected` on buttons per accessibility spec.
- Toggle changes mode, resets items to empty, resets cursor to null, sets hasMore to true, then immediately calls `loadMoreFeed(null, newMode)` via startTransition to fetch first page.
- Only show toggle if `followingCount > 0` (per D-01: before following anyone, only global mode).

Subtitle text below page title (inside this component):
- Global mode: `// ranked by rarity signal` (per D-06)
- Personal mode: `// signals from diggers you follow` (per D-06)
- `font-mono text-sm text-on-surface-variant`

Infinite scroll logic (per RESEARCH Pattern 1):
- `useEffect` watching `[inView, hasMore, isPending]`. When `inView && hasMore && !isPending`, call `startTransition(async () => { const newItems = await loadMoreFeed(cursor, mode); ... })`.
- Deduplicate: before appending, filter out items whose `id` already exists in current items.
- Update cursor to `newItems[newItems.length - 1].createdAt`.
- If `newItems.length === 0`, set `hasMore(false)`.

Rendering:
- Map over items: if `item.actionType === "added_record"` render `<FeedCard item={item} />`. If `item.actionType === "followed_user"` render `<FollowEventCard item={item} />`.
- After items: sentinel div `<div ref={sentinelRef} className="h-10" aria-hidden="true" />`
- Loading state: render 3 skeleton placeholders (simple `bg-surface-container-low rounded-lg h-48 animate-pulse` divs) when isPending.
- End of feed: `<div className="font-mono text-[10px] text-outline text-center py-8">[END_OF_FEED]</div>` when `!hasMore`.
- New items aria-live region: wrap the item list in `<div aria-live="polite">`.

Empty states (per UI-SPEC Copywriting):
