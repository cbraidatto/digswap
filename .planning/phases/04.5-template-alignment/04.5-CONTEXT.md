# Phase 4.5: Template Alignment — Context

**Gathered:** 2026-03-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove all GitHub metaphors from the existing UI, align navigation language with the DigSwap product vision, clean all hardcoded placeholder data, and fix broken button destinations. This phase does NOT add new features or backend — it is purely a UI/copy alignment pass on already-built surfaces.

Surfaces in scope:
- Left sidebar (shell)
- Bottom bar (shell)
- App header (shell)
- Feed page (/feed)
- Comunidade page (/comunidade)
- Perfil page (/perfil) — one button fix

</domain>

<decisions>
## Implementation Decisions

### Sidebar (Left Panel — Desktop)

- **D-01:** Keep the sidebar. Rename labels to match product language:
  - "Repositories" → "Collection" (link: /perfil)
  - "Organizations" → "Community" (link: /comunidade)
  - "Recent Activity" → "Feed" (link: /feed)
- **D-02:** Remove the "Popular Crews" section entirely — it contains 3 hardcoded fake crews (Jazz Diggers, Techno Underground, Ambient Archive). No replacement until Phase 7 builds real groups.
- **D-03:** The "NEW_COLLECTION" button in the sidebar header stays — it links to /perfil which is correct.

### Bottom Bar (Mobile)

- **D-04:** Remove the center FAB (Floating Action Button) entirely. It currently navigates to /trades/new which doesn't exist until Phase 9. No replacement — the bottom bar becomes a standard 4-tab bar: Feed · Explorar · Comunidade · Perfil.
- **D-05:** Tab labels and icons stay as-is (Feed, Explorar, Comunidade, Perfil).

### App Header

- **D-06:** Rename the desktop nav label "Digger" → "Explorar" to match the tab and route name.
- **D-07:** All other header elements stay unchanged (logo, XP display, notifications bell, avatar menu).

### Feed Page (/feed)

- **D-08:** Remove all 3 action buttons from feed items: Star, Fork, Comment. Feed items are read-only until Phase 5 builds the real social layer. No replacement buttons.
- **D-09:** Keep terminal-style status labels but rename to digger language:
  - `[NEW_COMMIT]` → `[NEW_FIND]`
  - `[RIP_SUCCESS]` → `[RIP]`
  - `[TRADE_OFFER]` → `[TRADE_OFFER]` (already correct)
- **D-10:** Remove the right sidebar entirely (currently shows "Trending_Repos" and "System_Logs" — both GitHub metaphors with fake data).
- **D-11:** Replace hardcoded FEED_ITEMS array with a clean empty state. The feed is empty until the user follows other diggers (Phase 5). Empty state copy should speak to diggers and hint at the future feature.
- **D-12:** Empty state for Feed: clean, honest, digger-flavored. No fake content. Example direction: terminal-style message indicating the feed is waiting for connections — "no signals yet / follow diggers to see their finds here."

### Comunidade Page (/comunidade)

- **D-13:** Remove all hardcoded fake data:
  - OPEN_SWAPS array (3 fake trade proposals)
  - GENRE_CREWS array (Breakcore_Front, Ambient_Voids, Acid_Labs, etc.)
  - Any hardcoded leaderboard/top diggers data
- **D-14:** Replace each tab content (Swaps, Discussions, Crews) with clean empty states. Each should:
  - Use terminal/digger-flavored copy
  - Hint at what the feature will be (not just "coming soon")
  - Match the Ghost Protocol visual theme (no white boxes, use surface colors)
- **D-15:** Tab structure (Swaps / Discussions / Crews) stays — just empty states replace fake content.

### Perfil Page (/perfil)

- **D-16:** The "ADD_RECORD" button that incorrectly links to /settings should be removed. The FAB (floating action button on the profile page) already handles adding records — no need for a duplicate broken link.

### Language & Tone

- **D-17:** Terminal/digger vocabulary is intentional and stays. The aesthetic is "Ghost Protocol" — dark, monospace, terminal-style. What changes is the DOMAIN — swap GitHub metaphors for vinyl/digging metaphors.
  - GitHub words to remove: commit, fork, star, repository, organization, merge, branch
  - Digger words to use: find, dig, rip, trade, collection, crew, drop, press(ing)

</decisions>

<specifics>
## Specific Notes

- The sidebar "NEW_COLLECTION" button label is fine as-is.
- Status label colors (primary/secondary/tertiary) can stay — just the text content changes.
- Feed item card structure (cover art placeholder, artist/title/genre/label metadata display) is good and stays — only the action buttons and status labels change.
- The right sidebar on /feed (2xl breakpoint, fixed position) is removed entirely — no replacement.
- Bottom bar goes from 5 elements (Feed · Explorar · FAB · Comunidade · Perfil) to 4 tabs (Feed · Explorar · Comunidade · Perfil). Spacing should be even across 4 tabs.
- Empty states must use the existing design system: surface-container-low backgrounds, primary green accents, font-mono for terminal copy, material-symbols-outlined icons.

</specifics>
