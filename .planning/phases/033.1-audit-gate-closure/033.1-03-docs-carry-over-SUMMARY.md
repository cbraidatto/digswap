## Decisions Made

- **Task 3 (checkpoint) resolution captured as evidence/03c attestation rather than empty commit.** Rationale: an empty commit carries no audit value. A 33-line attestation file in `evidence/` records *what was verified*, *who verified it*, *when*, and *why* the verification couldn't be self-service from Claude (gitignore boundary). Future auditors can read the file and reconstruct the decision context.
- **Annotation format `<!-- PASS in Phase 33 (audit drift fix 2026-04-24, Phase 33.1) -->` for the 3 drift-fix flips.** Rationale: the audit-drift framing is more accurate than "closed in 33.1" because the work was always PASS — only the checkbox state drifted. The annotation preserves the historical truth.
- **Traceability table edited in-place rather than via `requirements mark-complete`.** Rationale: the project's `gsd-tools requirements mark-complete` command flips the checkbox but doesn't update the Traceability table (separate structure). Direct Edit was the simplest path and matches the pattern Plans 01/02/04 used implicitly when their box flips were tool-driven but their Traceability rows were untouched until this plan.
