====================================================================

Executed:    2026-04-23T18:35:46.226Z

Background: app calls admin.rpc('vault_create_secret', ...). PostgREST
maps rpc names to public schema. Supabase ships vault.create_secret in
the vault schema, NOT public. We need to verify whether public.vault_create_secret
exists as a wrapper.

Query 1: search for any vault_create_secret / create_secret functions across schemas
SELECT n.nspname, p.proname, pg_get_function_identity_arguments(p.oid) AS args
FROM pg_proc p JOIN pg_namespace n ON n.oid = p.pronamespace
WHERE p.proname IN ('vault_create_secret', 'create_secret', 'vault_delete_secret', 'vault_update_secret')
ORDER BY n.nspname, p.proname;

Result rows: 1
[
  {
    "nspname": "vault",
    "proname": "create_secret",
    "args": "new_secret text, new_name text, new_description text, new_key_id uuid"
  }
]

hasPublicWrapper (public.vault_create_secret):  false
hasVaultNative   (vault.create_secret):         true

Hypothesis C CONFIRMED: app's admin.rpc('vault_create_secret', ...) call
fails because no public.vault_create_secret wrapper exists. PostgREST returns
a 404-style error object that the silent try/catch in oauth.ts swallows.
