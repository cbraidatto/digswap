# Plan 17-03: Lease RPC + Heartbeat + Reconciliation Receipts + ICE Telemetry

## Goal
Add the runtime authority layer for DigSwap Desktop: database-backed trade leases, authenticated handoff-token exchange, desktop heartbeat + release behavior, and local receipt reconciliation with ICE telemetry.

## Context
This plan sits between the Electron shell (`17-02`) and the full lobby/transfer runtime (`17-06`). The renderer inbox from `17-04` is already integrated, so `getPendingTrades()` and `openTradeFromHandoff()` should stop being placeholders in this phase even if the visual lobby itself still lands in `17-06`.

## Tasks

### Task 1: Add database surface for runtime authority and reconciliation

**Files:**
- `apps/web/src/lib/db/schema/trades.ts`
- `supabase/migrations/20260331_desktop_trade_runtime.sql`

**Steps:**

1. Add new tables to the trade schema:
   - `trade_runtime_sessions`
   - `trade_transfer_receipts`
2. Capture:
   - `trade_id`
   - `user_id`
   - `device_id`
   - `client_kind`
   - `desktop_version`
   - `trade_protocol_version`
   - lease timestamps
   - `last_ice_candidate_type`
3. Add SQL RPCs:
   - `list_desktop_pending_trades`
   - `acquire_trade_lease`
   - `heartbeat_trade_lease`
   - `release_trade_lease`
   - `finalize_trade_transfer`
4. Keep direct writes blocked; use RPCs for authority mutation.

### Task 2: Add authenticated web handoff token exchange for desktop

**Files:**
- `apps/web/src/app/api/desktop/handoff/consume/route.ts`

**Steps:**

1. Accept `POST` with `{ tradeId, token }`.
2. Authenticate the desktop caller from `Authorization: Bearer <accessToken>`.
3. Use the existing handoff-token infrastructure to verify + consume the token exactly once.
4. Return a minimal JSON payload the desktop main process can trust to continue lease acquisition.

### Task 3: Implement desktop trade runtime service

**Files:**
- `apps/desktop/src/main/trade-runtime.ts`
- `apps/desktop/src/main/session-store.ts`
- `apps/desktop/src/main/supabase-auth.ts`
- `apps/desktop/src/main/config.ts`

**Steps:**

1. Add a persistent desktop `deviceId`.
2. Implement a main-process runtime service that:
   - fetches pending trades from RPC
   - exchanges web handoff tokens through the new API route
   - acquires/releases a lease via RPC
   - runs a 15s heartbeat loop
   - stores pending transfer receipts locally
   - retries receipt reconciliation on startup and on auth/session availability
3. Record ICE candidate type in both the local receipt queue and the finalize RPC payload.

### Task 4: Wire IPC to the runtime without breaking the renderer contract

**Files:**
- `apps/desktop/src/main/ipc.ts`
- `apps/desktop/src/main/index.ts`
- `apps/desktop/src/shared/ipc-types.ts`

**Steps:**

1. Replace the placeholder `getPendingTrades()` implementation with real RPC-backed data.
2. Replace the throwing `openTradeFromHandoff()` implementation with real handoff/lease acquisition.
3. Keep the existing renderer contract stable for now.
4. Compatibility shim: until the contract is renamed in a later phase, `PendingTrade.handoffToken` may carry a trade-id string for inbox-open actions initiated entirely inside desktop.
