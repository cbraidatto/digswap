<objective>
Web handoff page + download gate + version gate for the desktop trade runtime.

Purpose: When a user initiates a trade from the web app, they must be transferred to the Electron desktop app. This plan implements the full handoff chain: CTA button generates a short-TTL signed token, the web intermediary page fires the protocol handler, detects if the app is installed within 1.5s, shows OS-specific download links if not, and blocks outdated clients with an upgrade prompt.

Output: `src/lib/desktop/handoff-token.ts`, `src/actions/desktop.ts`, `src/app/desktop/open/` (page + client component), `src/components/trade/open-in-desktop-button.tsx`, and a schema migration adding the `handoff_tokens` table.
</objective>

       ```
       export const handoffTokens = pgTable("handoff_tokens", {
         id: uuid("id").defaultRandom().primaryKey(),
         tradeId: uuid("trade_id").notNull().references(() => tradeRequests.id, { onDelete: "cascade" }),
         userId: uuid("user_id").notNull(),
         tokenHmac: varchar("token_hmac", { length: 64 }).notNull(), // hex-encoded HMAC-SHA256
         expiresAt: timestamp("expires_at", { withTimezone: true }).notNull(),
         usedAt: timestamp("used_at", { withTimezone: true }),
         createdAt: timestamp("created_at", { withTimezone: true }).defaultNow().notNull(),
       }, (table) => [
         pgPolicy("handoff_tokens_no_direct_access", {
           for: "all",
           to: authenticatedRole,
           using: sql`false`, // all access via admin client (server actions only)
         }),
       ]);
       ```
       Also export from src/lib/db/schema/index.ts.

    2. Create src/lib/desktop/handoff-token.ts:
       - Use `import { createHmac, randomBytes, timingSafeEqual } from "node:crypto"` — these are available in Next.js Node.js runtime (not Edge; the desktop handoff route uses Node.js runtime)
       - HMAC_SECRET from process.env.HANDOFF_HMAC_SECRET — throw if missing
       - createHandoffToken(tradeId: string, userId: string): Promise<string>
         - generates 32 bytes via randomBytes(32).toString("hex") → plaintext
         - computes hmac = createHmac("sha256", HMAC_SECRET).update(plaintext).digest("hex")
         - inserts into handoffTokens: { tradeId, userId, tokenHmac: hmac, expiresAt: new Date(Date.now() + 300_000) }
         - returns plaintext
       - verifyAndConsumeHandoffToken(plaintext: string, tradeId: string, userId: string): Promise<boolean>
         - recompute hmac from plaintext
         - fetch row WHERE tokenHmac = hmac AND tradeId = tradeId AND userId = userId AND usedAt IS NULL
         - if not found or expiresAt < now() → return false
         - UPDATE handoffTokens SET usedAt = now() WHERE id = row.id AND usedAt IS NULL (returns updated row count)
         - if update count === 0 → return false (concurrent replay attempt won)
         - return true
       - Use db (Drizzle) for all queries — NOT admin Supabase client (RLS is set to false/no-direct, server action bypasses via db directly)

    3. Create src/actions/desktop.ts:
       - "use server"
       - generateHandoffToken(tradeId: string): Promise<{ token: string } | { error: string }>
         - getUser() from createClient — return error if unauthenticated
         - fetch trade from tradeRequests WHERE id = tradeId AND (requesterId = userId OR providerId = userId)
         - return error if not found or not participant (IDOR protection)
         - call createHandoffToken(tradeId, userId)
         - return { token }
       - checkDesktopVersion(): Promise<{ minVersion: number }>
         - reads NEXT_PUBLIC_MIN_DESKTOP_VERSION (parseInt, default 1)
         - returns { minVersion }
         - Note: heartbeat version comparison is handled client-side in the open page using the token payload from the desktop app — this action just exposes the gate threshold
