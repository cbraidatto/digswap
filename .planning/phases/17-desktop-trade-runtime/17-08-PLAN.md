## Objective

Replace the synthetic file materialised in `ensureSourceFile()` with a real native file picker (`dialog.showOpenDialog`) so the sender path is production-ready. The receiver path is already solid. This is the final gap in Phase 17.

## Context

`trade-runtime.ts:807` — `ensureSourceFile()` writes a zero-filled buffer to `downloadPath/.digswap-outgoing/<tradeId>_<name>` and uses that as the source. This was an explicit placeholder left by 17-07. The WebRTC transport is real; only the source asset is fake.

The fix is narrow: swap `ensureSourceFile()` for `pickSourceFile()` that opens a native dialog and returns the chosen path. Everything downstream (`sendFile`, progress events, SHA-256, completion) stays identical.

**Replace `ensureSourceFile()` with `pickSourceFile()`:**

```typescript
private async pickSourceFile(context: CachedTradeContext): Promise<string> {
  const lastDir = this.sessionStore.getLastSourceDirectory();
  const defaultPath = lastDir ?? this.sessionStore.getSettings().downloadPath;

  const { canceled, filePaths } = await dialog.showOpenDialog({
    title: `Select vinyl rip to send — ${context.detail.myLeg.title}`,
    defaultPath,
    properties: ["openFile"],
    filters: [
      { name: "Audio Files", extensions: ["flac", "mp3", "wav", "aiff", "ogg", "m4a", "aac"] },
      { name: "All Files", extensions: ["*"] },
    ],
  });

  if (canceled || filePaths.length === 0) {
    throw new UserCancelledPickerError();
  }

  const chosen = filePaths[0];
  this.sessionStore.setLastSourceDirectory(path.dirname(chosen));
  return chosen;
}
```

private async runSenderTransfer(
