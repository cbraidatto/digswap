# Plan 17-02: Electron Shell + Auth Callback + Secure Token Storage + Protocol Handler

## Goal
Turn `apps/desktop` into a working Electron shell that can launch, register the `digswap://` protocol, start Supabase OAuth in the external browser, receive auth callbacks, persist auth state securely, and expose a typed preload bridge for the renderer.

## Context
Implements ADR-002 D-01, D-06, D-08, and D-09 on top of the 17-01 monorepo bootstrap and Claude's 17-05 web handoff flow. This plan intentionally focuses on shell/runtime plumbing so the renderer work in 17-04 can layer in without needing to redesign the main-process boundary.

## Tasks

### Task 1: Replace the placeholder package with a real Electron workspace

**Files:**
- `apps/desktop/package.json`
- `apps/desktop/tsconfig.json`
- `apps/desktop/electron.vite.config.ts`

**Steps:**

1. Convert `@digswap/desktop` into an Electron + React workspace package using:
   - `electron`
   - `electron-vite`
   - `vite`
   - `@vitejs/plugin-react`
   - `react` / `react-dom`
   - `@supabase/supabase-js`
   - `electron-store`
2. Replace placeholder scripts with:
   - `dev`
   - `build`
   - `preview`
   - `typecheck`
3. Update TypeScript config so main, preload, shared, and renderer all compile cleanly.
4. Add `electron.vite.config.ts` with separate `main`, `preload`, and `renderer` targets.

### Task 2: Implement secure main-process foundations

**Files:**
- `apps/desktop/src/main/index.ts`
- `apps/desktop/src/main/window.ts`
- `apps/desktop/src/main/protocol.ts`
- `apps/desktop/src/main/session-store.ts`
- `apps/desktop/src/main/ipc.ts`

**Steps:**

1. Create the Electron bootstrap with:
   - single-instance lock
   - protocol registration for `digswap://`
   - `open-url` handling on macOS
   - `second-instance` handling on Windows/Linux
2. Create a locked-down `BrowserWindow`:
   - `contextIsolation: true`
   - `nodeIntegration: false`
   - `sandbox: true`
   - preload script only
3. Implement protocol parsing for:
   - `digswap://trade/<tradeId>?handoff=<token>`
   - `digswap://auth/callback?code=<code>`
4. Queue pending deep-link payloads so the renderer can consume them after launch.
5. Add secure session persistence using `electron-store` with values encrypted via `safeStorage`.
   - Persist a generic key/value vault so Supabase PKCE and session keys can live in the same encrypted store.
   - Report storage backend health to the renderer, including the Linux `basic_text` caveat when relevant.

### Task 3: Implement Supabase desktop auth runtime

**Files:**
- `apps/desktop/src/main/supabase-auth.ts`
- `apps/desktop/src/shared/ipc.ts`
- `apps/desktop/src/preload/index.ts`

**Steps:**

1. Create a main-process Supabase client configured for:
   - PKCE flow
   - custom storage adapter backed by the secure session store
   - `detectSessionInUrl: false`
2. Add `startOAuthSignIn(provider)` that:
   - requests an OAuth URL from Supabase
   - uses `digswap://auth/callback` as redirect target
   - opens the returned URL in the external browser
3. Add callback exchange handling that:
   - processes the `code` from `digswap://auth/callback`
   - calls `exchangeCodeForSession(code)` in the main process
   - notifies the renderer of auth state changes or callback failures
4. Expose a typed preload bridge for:
   - bootstrap state
   - auth session summary
   - start OAuth sign-in
   - sign out
   - deep-link event subscription

### Task 4: Add a minimal renderer shell for 17-04 to build on

**Files:**
- `apps/desktop/src/renderer/index.html`
- `apps/desktop/src/renderer/src/main.tsx`
- `apps/desktop/src/renderer/src/app.tsx`
- `apps/desktop/src/renderer/src/styles.css`

**Steps:**

1. Create a minimal React renderer that can boot inside the Electron shell.
2. Show:
   - app version
   - storage security status
   - auth state summary
   - latest deep-link payload
3. Add a basic "Sign in with Google" action through the preload bridge.
4. Keep the UI intentionally thin so 17-04 can replace it without merge pain.

## Verification
- [ ] `pnpm --dir apps/desktop exec tsc --noEmit` passes
- [ ] `pnpm --dir apps/desktop dev` starts the Electron shell without type/runtime bootstrap errors
- [ ] `apps/desktop` registers a typed preload API and does not expose Node directly to the renderer
- [ ] Deep-link parsing supports both `trade` handoff and `auth/callback`
- [ ] Secure session store uses `safeStorage` and does not persist plaintext session payloads

## Success Criteria
- Desktop app launches under Electron with a locked-down renderer boundary
- `digswap://` protocol handling works end-to-end inside the main process
- Supabase OAuth can be initiated externally and callback exchange is wired in the app shell
- Session persistence uses encrypted local storage via `safeStorage`
- Claude can build renderer screens on top of the preload contracts without changing shell architecture
