# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation + Authentication.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Aesthetic directive:** "Like a low-light vinyl record shop" — warm, textured, atmospheric. Not sterile.
> Every frontend component MUST follow the Anthropic Claude Cookbook "Distilled Aesthetics" prompting methodology.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (to be initialized at project scaffolding) |
| Preset | Custom dark-warm preset (CSS variables defined below) |
| Component library | Radix (via shadcn/ui) |
| Icon library | Lucide React (default with shadcn/ui) |
| Display font | Fraunces (Google Fonts, variable, optical sizing) |
| Body font | DM Sans (Google Fonts, variable) |
| Styling | Tailwind CSS v4 with @theme directives + CSS variables |
| Color format | OKLCH (Tailwind v4 native) |

**Note:** Project is greenfield. `components.json` will be created when Next.js project is scaffolded. Initialize shadcn/ui with `npx shadcn@latest init` and apply the CSS variable overrides from the Color section below.

### shadcn Components Required (Phase 1)

Install these components during scaffolding:

```
button input label card form separator avatar skeleton badge sonner dialog alert
```

### Font Loading

Load via `next/font/google` in the root layout:

```
Fraunces: weights 400, 600; subsets latin; display swap; variable --font-fraunces
DM Sans: weights 400, 600; subsets latin; display swap; variable --font-dm-sans
```

Apply `font-dm-sans` as the default body font. Apply `font-fraunces` on headings and display text only.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight label-to-input |
| sm | 8px | Compact element spacing, form field internal padding |
| md | 16px | Default element spacing, card padding, input padding |
| lg | 24px | Section padding inside cards, form group spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page section vertical spacing |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets on buttons: minimum 44px height (WCAG 2.5.8 target size)
- Auth form cards: 24px internal padding on mobile, 32px on desktop (responsive)

## Typography

| Role | Font | Size | Weight | Line Height | Letter Spacing |
|------|------|------|--------|-------------|----------------|
| Body | DM Sans | 16px | 400 (regular) | 1.5 | 0 |
| Label | DM Sans | 14px | 400 (regular) | 1.4 | 0.01em |
| Heading | Fraunces | 24px | 600 (semibold) | 1.2 | -0.02em |
| Display | Fraunces | 36px (desktop) / 28px (mobile) | 400 (regular) | 1.1 | -0.03em |

**Rules:**
- Maximum 2 font families in the entire phase (Fraunces + DM Sans)
- Maximum 2 weights across both families (400, 600)
- Body text MUST be 16px minimum — no 14px body copy, no 12px body copy
- Labels at 14px are permitted only for form labels and metadata captions
- Headings use Fraunces with negative letter-spacing for the tight, warm editorial feel
- Display size: 36px on desktop (>=480px), 28px on mobile (<480px). Reserved for page titles only: "Sign In", "Create Account", "Welcome to VinylDig"

## Color

All colors defined in OKLCH for Tailwind v4 compatibility.

### Core Palette

| Role | OKLCH | Hex Approx | Usage |
|------|-------|------------|-------|
| Dominant (60%) — Background | `oklch(0.13 0.02 60)` | #0D0B09 | Page background, root surface |
| Secondary (30%) — Surface | `oklch(0.18 0.02 55)` | #1A1714 | Cards, form containers, elevated surfaces |
| Accent (10%) — Amber | `oklch(0.72 0.14 65)` | #D4872C | Primary CTA buttons, active form focus rings, links, active nav indicators |
| Destructive | `oklch(0.55 0.16 25)` | #C44133 | Delete account, destructive confirmations only |

### Extended Tokens

| Token | OKLCH | Hex Approx | Usage |
|-------|-------|------------|-------|
| `--foreground` | `oklch(0.90 0.01 70)` | #E8E0D4 | Primary text (warm off-white) |
| `--muted-foreground` | `oklch(0.60 0.03 60)` | #8A8279 | Secondary text, placeholders, captions |
| `--border` | `oklch(0.25 0.02 55)` | #2E2924 | Card borders, dividers, input borders |
| `--ring` | `oklch(0.72 0.14 65)` | #D4872C | Focus rings (matches accent) |
| `--input` | `oklch(0.20 0.02 55)` | #1F1B17 | Input field backgrounds |
| `--card` | `oklch(0.18 0.02 55)` | #1A1714 | Card backgrounds (matches secondary) |
| `--popover` | `oklch(0.18 0.02 55)` | #1A1714 | Popover/dialog backgrounds |
| `--success` | `oklch(0.65 0.14 145)` | #4D8C3F | Email verified confirmation, success toasts |
| `--warning` | `oklch(0.75 0.14 85)` | #C4A035 | Session limit warnings, 2FA suggestion |

### Accent Reserved For (explicit list)

The accent color (#D4872C amber) is ONLY used on:
1. Primary CTA buttons ("Create Account", "Sign In", "Continue Setup")
2. Text links on dark backgrounds
3. Active focus ring on form inputs
4. Loading spinner/progress indicator
5. The VinylDig wordmark/logo accent

Accent is NOT used on: backgrounds, cards, borders (except focus rings), body text, headings, icons in resting state.

### Grain Texture

Apply a subtle CSS noise grain on the dominant background surface:

```css
.grain::after {
  content: "";
  position: fixed;
  inset: 0;
  opacity: 0.03;
  pointer-events: none;
  z-index: 50;
  background-image: url("data:image/svg+xml,%3Csvg viewBox='0 0 256 256' xmlns='http://www.w3.org/2000/svg'%3E%3Cfilter id='noise'%3E%3CfeTurbulence type='fractalNoise' baseFrequency='0.9' numOctaves='4' stitchTiles='stitch'/%3E%3C/filter%3E%3Crect width='100%25' height='100%25' filter='url(%23noise)'/%3E%3C/svg%3E");
}
```

- Opacity: 0.03 (barely visible, adds texture without interfering with readability)
- Applied to the root layout wrapper, NOT to individual cards or components
- Uses SVG feTurbulence filter (no external image dependency)
- `pointer-events: none` so it does not interfere with interaction

### Layout Pattern

All auth pages share a single layout:
- Full viewport height, flex column centered
- Dominant background with grain texture
- VinylDig wordmark above the form card (Fraunces 36px, 400 weight)
- Form card: secondary surface, 1px border, 8px border-radius, 24px padding (mobile) / 32px padding (desktop)
- Below card: auxiliary links (small text, muted-foreground)

### Interaction States

| Element | Resting | Hover | Focus | Disabled | Loading |
|---------|---------|-------|-------|----------|---------|
| Primary button | Accent bg, dominant text | Accent lightened 10% | 2px accent ring, 2px offset | 50% opacity, no pointer | Spinner replaces label, same width |
| Secondary button | Transparent bg, border | Border lightens to muted-foreground | 2px accent ring, 2px offset | 50% opacity | Spinner replaces label |
| Text input | Input bg, border, foreground text | Border lightens | Accent ring, border hidden | 40% opacity bg | N/A |
| Text link | Accent color, no underline | Underline appears | 2px accent ring | N/A | N/A |
| Social login button | Secondary bg, border, icon + label | Border lightens, subtle bg shift | 2px accent ring | 50% opacity | Spinner replaces icon |

### Form Validation

- Inline validation: error messages appear below the input field
- Error text: `--destructive` color, 14px DM Sans, 400 weight
- Error border: input border changes to `--destructive` color
- Success indicator: green checkmark icon appears inside the input (right side) for email format validation
- All validation messages must be associated with inputs via `aria-describedby`

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Color contrast (WCAG AA) | Foreground (#E8E0D4) on dominant (#0D0B09) = 14.5:1. Foreground on secondary (#1A1714) = 11.2:1. Both exceed 4.5:1 AA minimum. |
| Focus indicators | 2px solid accent ring with 2px offset on all interactive elements. Visible on both keyboard and mouse focus. |
| Form labels | Every input has a visible `<label>` element. No placeholder-only inputs. |
| Error association | All error messages use `aria-describedby` linked to their input. |
| Button states | Disabled buttons use `aria-disabled="true"` and reduced opacity. Loading buttons use `aria-busy="true"`. |
| Skip navigation | Not required in Phase 1 (no navigation chrome). Add in Phase 2. |
| Reduced motion | Respect `prefers-reduced-motion: reduce`. Disable grain animation and button transitions. |
| Screen reader flow | Auth forms use semantic `<form>`, `<fieldset>`, `<legend>` elements. Social login section uses `<section aria-label="Social login">`. |

| Page | Loading State |
|------|---------------|
| Auth pages (sign in, sign up) | Full card skeleton: gray shimmer blocks matching form layout (inputs + button). 3 skeleton lines for inputs, 1 skeleton button. |

| 2FA QR code | Gray 200x200px skeleton square in place of QR code. Text content loads immediately. |
| Password reset | Same as auth pages skeleton pattern. |

Skeleton shimmer: use CSS `@keyframes shimmer` with a subtle warm gradient sweep (not cold gray). Base: `--border` color. Sweep: 5% lighter than base.

## CSS Variable Map

These CSS variables must be set in the root stylesheet when shadcn is initialized. They override shadcn defaults for the dark-warm theme.

```css
:root {
  /* VinylDig Dark Warm Theme */
  --background: oklch(0.13 0.02 60);
  --foreground: oklch(0.90 0.01 70);
  --card: oklch(0.18 0.02 55);
  --card-foreground: oklch(0.90 0.01 70);
  --popover: oklch(0.18 0.02 55);
  --popover-foreground: oklch(0.90 0.01 70);
  --primary: oklch(0.72 0.14 65);
  --primary-foreground: oklch(0.13 0.02 60);
  --secondary: oklch(0.20 0.02 55);
  --secondary-foreground: oklch(0.90 0.01 70);
  --muted: oklch(0.20 0.02 55);
  --muted-foreground: oklch(0.60 0.03 60);
  --accent: oklch(0.72 0.14 65);
  --accent-foreground: oklch(0.13 0.02 60);
  --destructive: oklch(0.55 0.16 25);
  --destructive-foreground: oklch(0.95 0.01 70);
  --border: oklch(0.25 0.02 55);
  --input: oklch(0.20 0.02 55);
  --ring: oklch(0.72 0.14 65);
  --success: oklch(0.65 0.14 145);
  --success-foreground: oklch(0.13 0.02 60);
  --warning: oklch(0.75 0.14 85);
  --warning-foreground: oklch(0.13 0.02 60);
  --radius: 0.5rem;
}
```

**Important:** Since this is dark-only (D-01), there is no `.dark` class or `prefers-color-scheme` media query. These values are the only values. No light mode variables exist.
