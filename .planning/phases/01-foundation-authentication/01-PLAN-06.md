<task type="auto">
  <name>Task 1: Create backup code utilities and MFA server actions</name>
  <files>src/lib/backup-codes.ts, src/actions/mfa.ts</files>
  <action>
    1. **src/lib/backup-codes.ts** -- Custom backup code implementation (per 01-RESEARCH.md Pitfall 6, AUTH-06):
       - `generateBackupCodes(count = 10)`: Generate `count` random alphanumeric codes (8 characters each, uppercase). Use crypto.randomBytes for secure randomness. Return array of plaintext codes.
       - `hashBackupCode(code: string)`: Hash a backup code using bcrypt (work factor 10). Return the hash. Install `bcryptjs` (pure JS, no native dependency) if not already available, or use the Node.js crypto module with scrypt.
       - `verifyBackupCode(code: string, hash: string)`: Compare plaintext code against bcrypt hash. Return boolean.
       - `storeBackupCodes(userId: string, codes: string[])`: Hash all codes and insert into backup_codes table. Delete any existing codes for the user first (re-enrollment replaces old codes).
       - `consumeBackupCode(userId: string, code: string)`: Find all unused codes for user, verify against each hash, if match found mark as used (set `used = true`, `used_at = now()`). Return boolean indicating success.

    2. **src/actions/mfa.ts** -- Server Actions for MFA (per 01-RESEARCH.md Pattern 3):
       - `enrollTotp()`: "use server"
         - Call `supabase.auth.mfa.enroll({ factorType: 'totp', friendlyName: 'VinylDig' })`
         - Generate 10 backup codes via `generateBackupCodes()`
         - Store hashed backup codes in database via `storeBackupCodes(userId, codes)`
         - Return `{ factorId, qrCode (SVG), uri, backupCodes (plaintext, shown once) }`
       - `verifyTotpEnrollment(factorId: string, code: string)`: "use server"
         - Rate limit by user ID using totpRateLimit
         - Validate code with totpSchema
         - Call `supabase.auth.mfa.challenge({ factorId })`
         - Call `supabase.auth.mfa.verify({ factorId, challengeId, code })`
         - On success: update profiles table to set `two_factor_enabled = true`
         - Return success/error
       - `challengeTotp(code: string)`: "use server"
         - Rate limit by user ID
         - Get user's TOTP factors via `supabase.auth.mfa.listFactors()`
         - Call `challenge()` then `verify()` with the first verified TOTP factor
         - On success: return `{ success: true, redirectTo: "/onboarding" or "/" }`
       - `useBackupCode(code: string)`: "use server"
         - Rate limit by user ID
         - Call `consumeBackupCode(userId, code)` from backup-codes.ts
         - If valid: create a Supabase session at AAL2 level (this may require using the admin client to issue a session, or alternatively bypassing the MFA challenge)
         - Return success/error. If all backup codes consumed, warn user to re-enroll
       - `disableTotp()`: "use server"
         - Requires current AAL2 level (user must have verified with TOTP first)
         - Get factors, call `supabase.auth.mfa.unenroll({ factorId })`
         - Update profiles table to set `two_factor_enabled = false`
         - Delete remaining backup codes for user
         - Return success
  </action>

  </verify>
  <done>Backup code utilities generate, hash, and verify codes securely. MFA server actions handle enrollment, verification, challenge, backup code usage, and disable. All actions are rate-limited.</done>
