<task type="auto">
  <name>Task 1: Create OAuth callback route and password reset server actions</name>
  <files>src/app/api/auth/callback/route.ts, src/actions/auth.ts</files>
  <action>
    1. **src/app/api/auth/callback/route.ts** -- OAuth and email verification callback (per 01-RESEARCH.md OAuth Callback example, D-09):
       - Export GET handler
       - Extract `code` and `next` from URL searchParams
       - Default `next` to `/onboarding` (per D-06, new users go to onboarding)
       - If code exists, call `supabase.auth.exchangeCodeForSession(code)`
       - On success: redirect to `${origin}${next}`
       - On error: redirect to `/signin?error=auth_callback_failed`
       - This handles BOTH OAuth callbacks (Google/GitHub) AND email verification link callbacks (PKCE flow)

    2. **src/actions/auth.ts** -- Add password reset server actions (append to existing file from Plan 04):
       - `forgotPassword(formData: FormData)`:
         - Rate limit by email using `resetRateLimit` (3 per 15 min per email, per D-16)
         - Validate with `forgotPasswordSchema`
         - Call `supabase.auth.resetPasswordForEmail(email, { redirectTo: "${SITE_URL}/reset-password" })`
         - ALWAYS return success message regardless of whether email exists (OWASP -- never reveal if account exists)
         - Return: `{ success: true, message: "Check your inbox. We sent a reset link to {email}." }` (per UI-SPEC Copywriting)
       - `resetPassword(formData: FormData)`:
         - Rate limit by IP using `authRateLimit`
         - Validate with `resetPasswordSchema` (enforces same strength as signup per D-18)
         - Call `supabase.auth.updateUser({ password: parsed.data.password })`
         - On success: return `{ success: true, message: "Password updated. You can now sign in with your new password." }` (per UI-SPEC)
         - On error: return generic error
  </action>

  </verify>
