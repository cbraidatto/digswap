<task type="auto">
  <name>Task 2: Define complete application schema with RLS policies</name>
  <files>src/lib/db/schema/users.ts, src/lib/db/schema/sessions.ts, src/lib/db/schema/collections.ts, src/lib/db/schema/releases.ts, src/lib/db/schema/social.ts, src/lib/db/schema/trades.ts, src/lib/db/schema/notifications.ts, src/lib/db/schema/gamification.ts, src/lib/db/schema/subscriptions.ts, src/lib/db/schema/index.ts</files>
  <action>
    Per D-10 (full app schema in Phase 1), D-11 (Drizzle ORM), D-12 (RLS from the start), create all tables using Drizzle's Supabase helpers (`authenticatedRole`, `authUid` from `drizzle-orm/supabase`) and `pgPolicy` from `drizzle-orm/pg-core`.

    Use this import pattern for all schema files:
    ```typescript
    import { pgTable, uuid, varchar, text, timestamp, boolean, integer, real, pgEnum } from "drizzle-orm/pg-core";
    import { pgPolicy } from "drizzle-orm/pg-core";
    import { sql } from "drizzle-orm";
    import { authenticatedRole, authUid } from "drizzle-orm/supabase";
    ```

    **users.ts** -- User profiles and auth-related data:
    - `profiles` table: id (uuid PK, references auth.users.id), display_name (varchar 50), avatar_url (text), bio (text), discogs_username (varchar 100), discogs_connected (boolean default false), onboarding_completed (boolean default false), two_factor_enabled (boolean default false), subscription_tier (varchar 20 default "free"), created_at, updated_at
    - RLS: SELECT for all authenticated users (public profiles). UPDATE/INSERT only own row (`id = auth.uid()`). DELETE own row only.

    **sessions.ts** -- Custom session tracking (D-13) and backup codes (AUTH-06):
    - `user_sessions` table: id (uuid PK), user_id (uuid not null), session_id (text not null unique -- Supabase session ID), device_info (text -- User-Agent), ip_address (text), created_at, last_seen_at
    - RLS: SELECT/DELETE only own sessions (`user_id = auth.uid()`). INSERT for authenticated role.
    - `backup_codes` table: id (uuid PK), user_id (uuid not null), code_hash (text not null -- bcrypt hash), used (boolean default false), used_at (timestamp nullable), created_at
    - RLS: SELECT/UPDATE only own codes (`user_id = auth.uid()`). INSERT for authenticated role. No DELETE (codes are invalidated, not deleted).

    **collections.ts** -- User record collections:
    - `collection_items` table: id (uuid PK), user_id (uuid not null), release_id (uuid references releases), discogs_instance_id (integer nullable), condition_grade (varchar 10 -- Mint/VG+/VG/G+/G/F/P), notes (text), added_via (varchar 20 -- "discogs"/"manual"), created_at, updated_at
    - RLS: SELECT for all authenticated (public collections). INSERT/UPDATE/DELETE own items only.

    **releases.ts** -- Record/release catalog data:
    - `releases` table: id (uuid PK), discogs_id (integer unique nullable), title (varchar 500 not null), artist (varchar 500 not null), year (integer), genre (text[] -- array of genres), style (text[] -- array of styles), country (varchar 100), format (varchar 100 -- LP/CD/7"/etc), label (varchar 500), cover_image_url (text), discogs_have (integer default 0), discogs_want (integer default 0), rarity_score (real -- computed from have/want ratio), created_at, updated_at
    - RLS: SELECT for all authenticated. INSERT/UPDATE for service role only (populated by import pipeline).

    **social.ts** -- Follow relationships and activity:
    - `follows` table: id (uuid PK), follower_id (uuid not null), following_id (uuid not null), created_at. Unique constraint on (follower_id, following_id).
    - `activity_feed` table: id (uuid PK), user_id (uuid not null), action_type (varchar 50 -- "added_record"/"wrote_review"/"completed_trade"/etc), target_type (varchar 50), target_id (uuid), metadata (jsonb), created_at
    - RLS: follows -- SELECT for all authenticated, INSERT/DELETE own follows only. Activity -- SELECT for all authenticated, INSERT own activities only.

    **trades.ts** -- P2P trade requests and history:
    - `trade_requests` table: id (uuid PK), requester_id (uuid not null), provider_id (uuid not null), release_id (uuid), status (varchar 20 -- "pending"/"accepted"/"declined"/"completed"/"cancelled"), message (text), created_at, updated_at
    - `trade_reviews` table: id (uuid PK), trade_id (uuid references trade_requests), reviewer_id (uuid not null), reviewed_id (uuid not null), quality_rating (integer 1-5), comment (text), created_at
    - RLS: trade_requests -- SELECT if requester or provider. INSERT for authenticated. UPDATE if requester or provider. trade_reviews -- SELECT for all authenticated. INSERT own reviews only.

    **notifications.ts** -- In-app and email notification tracking:
    - `notifications` table: id (uuid PK), user_id (uuid not null), type (varchar 50), title (text not null), body (text), link (text), read (boolean default false), created_at
    - `notification_preferences` table: id (uuid PK), user_id (uuid not null unique), wantlist_match_inapp (boolean default true), wantlist_match_email (boolean default true), trade_request_inapp (boolean default true), trade_request_email (boolean default true), trade_completed_inapp (boolean default true), ranking_change_inapp (boolean default true), new_badge_inapp (boolean default true), push_enabled (boolean default false), created_at, updated_at
    - RLS: Both tables -- SELECT/UPDATE/INSERT own rows only.

    **gamification.ts** -- Rankings, badges, leaderboards:
    - `user_rankings` table: id (uuid PK), user_id (uuid not null unique), rarity_score (real default 0), contribution_score (real default 0), global_rank (integer), title (varchar 100 -- "Crate Digger"/"Wax Prophet"/etc), updated_at
    - `badges` table: id (uuid PK), slug (varchar 100 unique not null), name (varchar 200 not null), description (text), icon_url (text), created_at
    - `user_badges` table: id (uuid PK), user_id (uuid not null), badge_id (uuid references badges), earned_at (timestamp)
    - RLS: user_rankings -- SELECT for all authenticated, UPDATE own only. badges -- SELECT for all. user_badges -- SELECT for all, INSERT service role only.

    **subscriptions.ts** -- Stripe subscription tracking:
    - `subscriptions` table: id (uuid PK), user_id (uuid not null unique), stripe_customer_id (text unique), stripe_subscription_id (text unique), status (varchar 50 -- "active"/"cancelled"/"past_due"/etc), plan (varchar 50 -- "free"/"premium_monthly"/"premium_annual"), current_period_start (timestamp), current_period_end (timestamp), trades_this_month (integer default 0), trades_month_reset (timestamp), created_at, updated_at
    - RLS: SELECT own subscription only. INSERT/UPDATE via service role only (webhook-driven).

    **wantlist.ts** -- User wantlist:
    - `wantlist_items` table: id (uuid PK), user_id (uuid not null), release_id (uuid references releases), notes (text), priority (integer default 0), created_at
    - RLS: SELECT for all authenticated (public wantlists). INSERT/DELETE own items only.

    **reviews.ts** -- Record reviews:
    - `reviews` table: id (uuid PK), user_id (uuid not null), release_id (uuid references releases), rating (integer 1-5 not null), title (varchar 200), body (text), is_pressing_specific (boolean default false), pressing_details (text), created_at, updated_at
    - RLS: SELECT for all authenticated. INSERT own reviews only. UPDATE/DELETE own reviews only.

    **groups.ts** -- Community groups:
    - `groups` table: id (uuid PK), creator_id (uuid not null), name (varchar 200 not null), description (text), category (varchar 100 -- genre/era/region/style), visibility (varchar 20 default "public"), member_count (integer default 0), created_at, updated_at
    - `group_members` table: id (uuid PK), group_id (uuid references groups), user_id (uuid not null), role (varchar 20 default "member"), joined_at (timestamp)
    - `group_posts` table: id (uuid PK), group_id (uuid references groups), user_id (uuid not null), content (text not null), created_at, updated_at
    - RLS: groups -- SELECT for all authenticated. INSERT for authenticated. UPDATE/DELETE for creator only. group_members -- SELECT for all. INSERT/DELETE own membership only. group_posts -- SELECT for all members. INSERT for members only.

    **index.ts** -- Re-export everything:
    ```typescript
    export * from "./users";
    export * from "./sessions";
    export * from "./collections";
    export * from "./releases";
    export * from "./social";
    export * from "./trades";
    export * from "./notifications";
    export * from "./gamification";
    export * from "./subscriptions";
    export * from "./wantlist";
    export * from "./reviews";
    export * from "./groups";
    ```

    NOTE: All tables use `uuid("id").defaultRandom().primaryKey()` pattern. All timestamps use `{ withTimezone: true }` and `.defaultNow()`. Foreign key references use `.references(() => table.id)` syntax.
  </action>
