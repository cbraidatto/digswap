  <name>Task 1: Build session management actions and UI</name>
  <files>src/actions/sessions.ts, src/app/(protected)/settings/sessions/page.tsx, src/components/settings/session-list.tsx</files>
  <action>
    1. **src/actions/sessions.ts** -- Server Actions for session management (per D-13, D-14, D-15, 01-RESEARCH.md Pattern 2):
       - `getSessions()`: "use server"
         - Get current user via getClaims()
         - Query user_sessions table for all sessions belonging to user, ordered by last_seen_at DESC
         - Return array of sessions with: id, device_info (parsed into readable format -- e.g., "Chrome on macOS"), ip_address, created_at, last_seen_at, isCurrent (boolean, comparing with current session ID)
       - `terminateSession(sessionId: string)`: "use server"
         - Get current user via getClaims()
         - Verify the session belongs to the current user (prevent IDOR)
         - Call `adminClient.auth.admin.signOut(supabaseSessionId)` to invalidate the Supabase session
         - Delete the record from user_sessions table
         - Return success
       - `enforceSessionLimit(userId: string, newSessionId: string)`: "use server" (called from signIn action)
         - Count active sessions for userId in user_sessions table
         - If count >= 3: find the oldest session (by created_at)
         - Invalidate oldest via `adminClient.auth.admin.signOut(oldestSessionId)`
         - Delete oldest from user_sessions
         - Insert new session record
         - This is called internally by the signIn server action -- not exposed to client
       - `recordSession(userId: string, sessionId: string, request: Request)`: internal helper
         - Extract device info from User-Agent header
         - Extract IP from x-forwarded-for header
         - Insert into user_sessions table
         - Call enforceSessionLimit if needed

    2. **src/components/settings/session-list.tsx** -- Active sessions list ("use client", per D-15, UI-SPEC Destructive Actions):
       - Fetch sessions on mount via getSessions()
       - Render each session as a card/row showing: device info, IP address, last seen time, "Current session" badge for active session
       - Each non-current session has "End Session" button (destructive color per UI-SPEC)
       - Inline confirmation on terminate: "End this session? You will be signed out on that device." (per UI-SPEC Destructive Actions)
       - After terminate: refresh the list
       - Show session count: "Active sessions: {count}/3" (per D-13)

    3. **src/app/(protected)/settings/sessions/page.tsx** -- Sessions settings page:
       - Server Component
       - Auth check via getClaims(), redirect if not authenticated
       - Page title: "Active Sessions"
       - Contains SessionList component
       - Note: This page will be integrated into a full settings layout in Phase 2. For now, it's a standalone page.
