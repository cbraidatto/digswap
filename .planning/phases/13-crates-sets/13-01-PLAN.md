**Table: crates**
- id: uuid, defaultRandom, primaryKey
- userId: uuid, notNull (references auth.users via authUid in RLS — do NOT add FK constraint, same pattern as collectionItems.userId)
- name: varchar(100), notNull
- date: date("date") — session date (not timestamp — use `date` type from drizzle-orm/pg-core, import `date` as a named import). Defaults to `sql\`CURRENT_DATE\``
- sessionType: varchar("session_type", { length: 20 }).notNull().default("digging_trip") — values enforced by CHECK: `sql\`session_type IN ('digging_trip','event_prep','wish_list','other')\``... add this as a Drizzle `check()` constraint in the table config array
- createdAt: timestamp with tz, defaultNow, notNull
- updatedAt: timestamp with tz, defaultNow, notNull

**Table: crate_items**
- id: uuid, defaultRandom, primaryKey
- crateId: uuid, notNull, references(() => crates.id, { onDelete: "cascade" })
- userId: uuid, notNull (denormalized for simpler RLS — same value as crates.userId)
- releaseId: uuid, nullable, references(() => releases.id) — null allowed (item added before release exists in DB)
- discogsId: integer("discogs_id"), nullable — snapshot of the Discogs ID at add time
- title: varchar(255), nullable — snapshot of release title at add time
- artist: varchar(255), nullable — snapshot of artist at add time
- coverImageUrl: text("cover_image_url"), nullable — snapshot of cover at add time
- status: varchar("status", { length: 20 }).notNull().default("active") — CHECK constraint: `sql\`status IN ('active','found')\``
- createdAt: timestamp with tz, defaultNow, notNull

**Table: sets** (a set belongs to a crate)
- id: uuid, defaultRandom, primaryKey
- crateId: uuid, notNull, references(() => crates.id, { onDelete: "cascade" })
- userId: uuid, notNull (denormalized for RLS)
- eventDate: date("event_date"), nullable
- venueName: varchar("venue_name", { length: 200 }), nullable
- createdAt: timestamp with tz, defaultNow, notNull
- updatedAt: timestamp with tz, defaultNow, notNull

**Table: set_tracks** (ordered tracks within a set)
- id: uuid, defaultRandom, primaryKey
- setId: uuid, notNull, references(() => sets.id, { onDelete: "cascade" })
- crateItemId: uuid, notNull, references(() => crate_items.id, { onDelete: "cascade" })
- userId: uuid, notNull (denormalized for RLS)
- position: integer("position").notNull() — 1-based, contiguous integers recomputed on save
- createdAt: timestamp with tz, defaultNow, notNull

**RLS policies for all 4 tables**: Apply the private pattern (SELECT/INSERT/UPDATE/DELETE all gated by `userId = authUid`). Name policies with the table prefix: `crates_select_own`, `crate_items_select_own`, etc.

**File 4: `src/actions/crates.ts`**

All functions are `"use server"` async server actions. Each must:
1. Call `createClient()` from `@/lib/supabase/server` and verify auth
2. Validate inputs with the corresponding Zod schema
3. Perform the DB operation using `db` client
4. Return `{ success: true, data?: ... }` or `{ success: false, error: string }`

Actions:

- `createCrate(input)` — INSERT into crates, return `{ success, data: { crateId } }`
- `updateCrate(input)` — UPDATE crates WHERE id = input.id AND userId = user.id
- `deleteCrate(crateId)` — DELETE from crates WHERE id AND userId = user.id (cascade deletes items, sets, tracks)
- `addToCrate(input)` — INSERT into crate_items. Before inserting, verify the target crate belongs to the user (SELECT crates WHERE id = input.crateId AND userId = user.id — return error if not found)
- `moveToWantlist(crateItemId)` — Lookup crate_item (verify ownership via userId). INSERT into wantlist_items with `addedVia: "crate"` and the releaseId from the item. Then UPDATE crate_items SET status = 'found' WHERE id = crateItemId. Return `{ success, data: { wantlistItemId } }`.
- `moveToCollection(crateItemId)` — Same pattern as moveToWantlist but INSERT into collection_items with `addedVia: "crate"`. UPDATE crate_items SET status = 'found'.
- `markAsFound(crateItemId)` — UPDATE crate_items SET status = 'found' WHERE id AND userId = user.id
- `createSet(input)` — INSERT into sets, then bulk INSERT into set_tracks (one row per crateItemId in trackOrder, position = array index + 1). Verify all crateItemIds belong to the crateId before inserting.
- `updateSetTracks(input)` — DELETE existing set_tracks WHERE setId, then bulk INSERT new rows with recomputed positions (1, 2, 3...). Verify set ownership.
- `deleteSet(setId)` — DELETE from sets WHERE id AND userId = user.id (cascade deletes tracks)
