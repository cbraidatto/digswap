**File: `src/app/(protected)/crates/[id]/_components/set-builder-panel.tsx`**

Client component. Props: `crateId: string; items: CrateItemRow[]; onClose: () => void`.

State:
- `eventDate: string` — defaults to today
- `venueName: string` — defaults ""
- `selectedItemIds: Set<string>` — tracks which items are checked
- `trackOrder: string[]` — ordered array of crateItemIds (built from selectedItemIds, reordered by drag)
- `isSaving: boolean`

Layout: A panel (`bg-surface-container rounded-lg border border-outline-variant/20 p-4 space-y-4`) that mounts inline below the `[+ NEW_SET]` button.

Sections:
1. Header: `[SET_BUILDER]` label + close button (X icon)
2. Event metadata row: date input + venue name input side by side
3. Track picker: show all active crate items as checkboxes. Each row: checkbox + cover thumbnail + title + artist. Checking an item adds its id to selectedItemIds and appends to trackOrder.
4. Ordered track list (only shown when selectedItemIds.size > 0): uses `@dnd-kit/sortable` `SortableContext` with `verticalListSortingStrategy`. Each draggable row shows: grip icon (GripVertical from lucide-react), position number, title, artist. Drag updates `trackOrder` via `onDragEnd` handler (array reorder using arrayMove from `@dnd-kit/sortable`).
5. `[SAVE_SET]` button: calls `createSet({ crateId, eventDate: eventDate || null, venueName: venueName || null, trackOrder })`. On success: toast "Set saved" + `router.refresh()` + `onClose()`. Disabled when `selectedItemIds.size === 0` or `isSaving`.

DnD setup:
```tsx
import { DndContext, closestCenter, type DragEndEvent } from "@dnd-kit/core";
import { SortableContext, verticalListSortingStrategy, useSortable, arrayMove } from "@dnd-kit/sortable";
import { CSS } from "@dnd-kit/utilities";
```

Each sortable item uses `useSortable({ id: crateItemId })` and applies `transform` + `transition` from the hook to the row style.
