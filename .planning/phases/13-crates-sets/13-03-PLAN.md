**Step 0: Add `getUserCratesAction` to `src/actions/crates.ts`**

Add one new exported server action at the bottom of the existing file:

```typescript
export async function getUserCratesAction(): Promise<
  (CrateRow & { itemCount: number })[]
> {
  const supabase = await createClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return [];
  return getCrates(user.id);
}
```

Import `getCrates` from `@/lib/crates/queries` and `CrateRow` from `@/lib/crates/types`.

**File: `src/components/crates/add-to-crate-popover.tsx`**

Client component. Replicates the QuickNotePopover @base-ui/react Popover pattern exactly.

Props:
```typescript
interface AddToCratePopoverProps {
  releaseId: string | null;         // UUID from our DB (null if release not yet imported)
  discogsId: number | null;
  title: string | null;
  artist: string | null;
  coverImageUrl: string | null;
  children: React.ReactNode;        // Trigger content (the [ADD_TO_CRATE] button)
}
```

State:
- `open: boolean` — controls popover
- `crates: (CrateRow & { itemCount: number })[]` — loaded when popover opens
- `isLoading: boolean` — while fetching crates
- `isAdding: string | null` — crateId currently being added to (for loading state)
- `showInlineCreate: boolean` — toggles the inline [+ New crate] form
- `newCrateName: string` — value for the inline create input

On popover open (`useEffect` on `open`): call `getUserCratesAction()` to populate `crates`.

**Popover content structure** (className: `"w-72 p-4 bg-surface-container border-outline-variant/20"`):

1. Label: `<p className="font-mono text-[10px] text-primary tracking-[0.15em] mb-3">[ADD_TO_CRATE]</p>`

2. Loading state: while `isLoading`, show spinner + "Loading crates..."

3. Crate list (when crates.length > 0 and !showInlineCreate):
   - Each crate: `<button>` with `w-full flex items-center justify-between py-2 px-2 rounded hover:bg-surface-container-high transition-colors`
   - Left: crate name in `font-mono text-[11px] text-on-surface`
   - Right: item count in `font-mono text-[10px] text-on-surface-variant` — e.g. "3"
   - On click: call `addToCrate({ crateId: crate.id, releaseId, discogsId, title, artist, coverImageUrl })`. On success: toast "Added to [crate name]" + close popover. On error: toast.error(result.error).
   - Disabled while `isAdding !== null`
   - Show spinner on the row where `isAdding === crate.id`

4. No-crates state (crates.length === 0 and !showInlineCreate):
   - `font-mono text-[10px] text-on-surface-variant/70` — "No crates yet. Create one below."
   - Auto-show inline create: set `showInlineCreate = true` on load when crates.length === 0

5. Inline create form (shown when `showInlineCreate === true` OR when user clicks [+ New crate]):
   - Input: `font-mono text-[11px] bg-surface-container-low border-outline-variant/20 h-8 w-full` — placeholder "Crate name..."
   - `[CREATE + ADD]` button: `font-mono text-[10px] bg-primary-container text-on-primary-container h-7 w-full`
   - On click:
     1. Call `createCrate({ name: newCrateName, date: today, sessionType: "digging_trip" })`
     2. On success: call `addToCrate({ crateId: result.data.crateId, ... })`
     3. Toast "Created and added to [name]" + close popover
   - Cancel button (only if crates.length > 0): `[CANCEL]` text button to set `showInlineCreate = false`

6. Bottom separator + `[+ New crate]` option (only shown when crates.length > 0 and !showInlineCreate):
   - `<button className="w-full font-mono text-[10px] text-primary hover:underline mt-2 text-left">[ + New crate ]</button>`
   - On click: `setShowInlineCreate(true)`

Today's date for `createCrate`: `new Date().toISOString().split("T")[0]`.
