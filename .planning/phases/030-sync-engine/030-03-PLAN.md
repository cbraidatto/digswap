    **Per D-03: All collection queries must add `WHERE deleted_at IS NULL` filter.**

    **Step 1: Audit all files that query collection_items.**
    Run: `grep -rn "collectionItems\|collection_items" apps/web/src/ --include="*.ts" --include="*.tsx" | grep -v "schema\|test\|node_modules" | sort -u`

    This will identify every file that references collection_items. For each file that performs a SELECT query (not just imports the schema), add the deletedAt IS NULL filter.

    **Step 2: Update buildWhereConditions (choke point).**

    In `apps/web/src/lib/collection/queries.ts`, add `isNull(collectionItems.deletedAt)` to the conditions array as the SECOND condition (right after userId check). This covers all collection page queries automatically:

    ```typescript
    import { eq, gte, lt, or, ilike, ne, isNull } from "drizzle-orm";

    function buildWhereConditions(
      userId: string,
      filters: CollectionFilters,
      options?: { excludePrivate?: boolean },
    ) {
      const conditions = [
        eq(collectionItems.userId, userId),
        isNull(collectionItems.deletedAt),  // <-- ADD THIS (per D-03)
      ];
      // ... rest unchanged
    }
    ```

    **Step 3: Update gamification ranking queries.**

    In `apps/web/src/lib/gamification/ranking.ts`, find the ranking CTE that counts collection items. Add `AND deleted_at IS NULL` to the WHERE clause. Also add `AND r.discogs_id IS NOT NULL` to exclude local-only releases from rarity scoring (per D-04: leaderboard/rarity queries exclude local releases).

    Look for SQL like:
    ```sql
    SELECT ... FROM collection_items ci JOIN releases r ON ...
    ```
    Add:
    ```sql
    WHERE ci.deleted_at IS NULL AND r.discogs_id IS NOT NULL
    ```

    **Step 4: Update discovery/radar queries.**

    In `apps/web/src/lib/discovery/radar.ts`, find queries that join collection_items. Add `.where(isNull(collectionItems.deletedAt))` or `AND deleted_at IS NULL` to raw SQL.

    **Step 5: Update collection actions.**

    In `apps/web/src/actions/collection.ts`, find any direct queries on collection_items (e.g., addRecordToCollection, removeFromCollection). For the addRecordToCollection duplicate check, add `isNull(collectionItems.deletedAt)` — a soft-deleted item should not prevent re-adding.

    **Step 6: Update comparison queries.**

    In `apps/web/src/lib/social/comparison.ts`, add `deleted_at IS NULL` filter to the collection comparison query.

    **Step 7: Scan for any missed files.**

    After the above, re-run the grep from Step 1 and verify every SELECT query on collection_items includes the deletedAt filter. Files that only reference the schema definition or only do INSERTs do not need the filter.

    **Important:** Do NOT add the filter to:
    - The sync route itself (apps/web/src/app/api/desktop/library/sync/route.ts) — it needs to see soft-deleted items to clear deletedAt on recovery
    - Schema definition files
    - Migration files
    - Test files (they test specific scenarios including soft-deleted items)
