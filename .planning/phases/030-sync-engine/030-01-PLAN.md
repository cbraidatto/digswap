<objective>
Build the web-side sync infrastructure: schema migration for deletedAt, normalization functions, batch processor with dedup logic, the API route that desktop calls, and the pg_cron purge job for expired soft-deleted items.

Purpose: This is the server half of the sync engine. Desktop sends batches of track metadata here; the server deduplicates against existing Discogs releases, creates local-only releases for unmatched items, upserts collection items, handles soft-deletion, and returns releaseMappings so the desktop can track which albumKeys map to which web-side release IDs.

Output: Working POST /api/desktop/library/sync endpoint with full dedup, deletion, and release mapping support. pg_cron purge migration ready.
</objective>

    export function normalizeForDedup(value: string | null): string {
      if (!value) return "";
      return value
        .toLowerCase()
        .replace(/^(the|a|an)\s+/i, "")
        .replace(/[^\w\s]/g, "")
        .replace(/\s+/g, " ")

    }

    **1. Batch Processor (per D-01, D-02, D-03, D-04)**

    Create `apps/web/src/lib/sync/process-sync-batch.ts`:

    Interface for the request payload:
    ```typescript
    export interface TrackSyncPayload {
      localTrackId: string;
      filePath: string;
      artist: string | null;
      album: string | null;
      title: string | null;
      year: number | null;
      trackNumber: number | null;
      format: string;
      bitrate: number;
      sampleRate: number;
      duration: number;
      artistConfidence: "high" | "low";
      albumConfidence: "high" | "low";
    }

    export interface ReleaseMapping {
      albumKey: string;
      releaseId: string;
    }

    export interface SyncResponse {
      ok: boolean;
      synced: number;
      created: number;
      linked: number;
      deleted: number;
      dedupQueued: number;
      releaseMappings: ReleaseMapping[];
      errors: string[];
    }
    ```

    `processSyncBatch(userId: string, tracks: TrackSyncPayload[], deletedReleaseIds: string[]): Promise<SyncResponse>`

    Logic:
    1. **Group tracks by album key** using `makeAlbumKey(track.artist, track.album)`.
    2. **For each album group** (Stage 1 dedup per D-01):
       - Query `releases` table: `SELECT id, discogs_id FROM releases WHERE lower(artist) = ? AND lower(title) = ?` using normalized values.
       - If a release with `discogsId IS NOT NULL` is found: use that release ID (Discogs match -> increment `linked` counter).
       - If a release with `discogsId IS NULL` is found: use that existing local release ID.
       - If no release found: INSERT a new release via `createAdminClient()` with `{ title: album, artist: artist, year: year, discogs_id: null }`. Increment `created` counter.
       - IMPORTANT: Do NOT use upsert on discogs_id for local releases (NULL != NULL pitfall). Use select-then-insert.
       - **Collect the albumKey->releaseId mapping for each album group** into `releaseMappings` array.
    3. **For each track in the group**: Upsert a `collection_items` row via admin client:
       - Conflict resolution on `(user_id, release_id)` — if already exists, update `updatedAt`, `audioFormat`, `bitrate`, `sampleRate`, clear `deletedAt` to NULL (recovery case).
       - Fields: `userId`, `releaseId`, `addedVia: "local"`, `audioFormat: track.format`, `bitrate: track.bitrate`, `sampleRate: track.sampleRate`, `visibility: "not_trading"` (default), `deletedAt: null`.
    4. **Low-confidence items** (per D-01): Tracks with `artistConfidence: "low"` OR `albumConfidence: "low"` that did NOT match an existing Discogs release get counted in `dedupQueued`. (Actual Discogs search queue deferred to Phase 32.)
    5. **Handle deletedReleaseIds** (per D-03, deviation from D-02 documented in CONTEXT.md): For each UUID in `deletedReleaseIds`:
       - `UPDATE collection_items SET deleted_at = now() WHERE user_id = ? AND release_id IN (?) AND added_via = 'local'`.
    6. **Return** `SyncResponse` with all counters AND `releaseMappings` array. This is critical — the desktop stores these mappings to enable deletion flow (Blocker fix #2).
