## Typography

| Role | Size | Weight | Line Height | Font Family | Usage in Phase 8 |
|------|------|--------|-------------|-------------|-------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Inter (font-sans) | Leaderboard score values, badge descriptions |

- `text-[8px]` — micro labels (verified badges)

Source: Existing typography patterns in `perfil/page.tsx`, `explorar/page.tsx`

## Color

All colors reference CSS custom properties from the Ghost Protocol theme (default). The design contract works across all 4 themes (ghost, chrome, rust, indigo) via CSS variables.

| Role | CSS Variable | Ghost Protocol Value | Usage |
|------|-------------|---------------------|-------|
| Dominant (60%) | `--background` / `--surface` | #10141a | Page background, tab panel background |
| Secondary (30%) | `--surface-container-low` / `--surface-container-high` | #181c22 / #262a31 | Leaderboard cards, rank display cards, badge containers, stat cards |
| Accent — Primary (10%) | `--primary` | #6fdd78 (green) | Own rank highlight, rank #1 indicator, earned badge borders, active tab, score values for rarity |
| Accent — Secondary | `--secondary` | #aac7ff (blue) | Rank title text, "Class Status" label, contribution score accent, genre filter chips (active) |
| Accent — Tertiary | `--tertiary` | #ffb689 (orange) | Section subheaders ("Digging Activity"), leaderboard section labels, genre leaderboard header accent |
| Destructive | `--destructive` | #ffb4ab | Not used in Phase 8 (no destructive actions) |
| Muted text | `--on-surface-variant` | #becab9 | Secondary text in leaderboard rows, unearned badge text, pagination text |
| Border | `--outline-variant` | #3e4a3d | Card borders, leaderboard row separators, genre filter chip borders |

Accent reserved for:
- `--primary` (#6fdd78): Active tab underline, own rank row highlight border, earned badge bracket color, rarity score number
- `--secondary` (#aac7ff): Rank title text, contribution score number, follow count accent, genre filter active state
- `--tertiary` (#ffb689): Section header labels ("Rankings", "Badges"), leaderboard scope label

Source: `globals.css` Ghost Protocol theme, existing usage in `perfil/page.tsx`, `explorar/page.tsx`

### LeaderboardRow Specification

Layout: Single horizontal row, full width within container.
Height: 48px (h-12), vertically centered content.
Background: `bg-surface-container-low` with `border border-outline-variant/10 rounded-lg`.
Own-user highlight: `border-l-2 border-primary bg-surface-container-high/50`.

Content format (per D-06):
```
#[rank] . [username] . [title] . [score]pts
```

| Segment | Font | Size | Color | Alignment |
|---------|------|------|-------|-----------|
| `#[rank]` | font-mono | text-sm (14px) | text-on-surface | Left, w-12 fixed |
| `.` separator | font-mono | text-[10px] | text-outline | Inline |
| `[username]` | font-mono | text-[11px] | text-on-surface | Flex-1, truncate |
| `[title]` | font-mono | text-[10px] | text-secondary | Flex-shrink-0 |
| `[score]pts` | font-mono | text-sm (14px) | text-primary | Right-aligned, tabular-nums |

Row gap between items: 8px (gap-2).
Vertical spacing between rows: 4px (space-y-1).
No avatar on rows (per D-06).

### GenreFilter Specification

Layout: Horizontal row of chip buttons, `flex gap-2 overflow-x-auto scrollbar-none`.
First chip: "GLOBAL" (always present, default active).
Remaining chips: One per Discogs genre from `DISCOGS_GENRES` constant (Electronic, Jazz, Hip Hop, Rock, Soul, Latin, Classical, Funk / Soul, Reggae, Pop, Blues, Folk, Country, Stage & Screen, Brass & Military).

Active state: `bg-secondary/15 text-secondary border-secondary/30 font-semibold`.
Inactive state: `bg-transparent text-on-surface-variant border-outline-variant/20 hover:text-on-surface hover:border-outline-variant/40`.
Chip sizing: `px-3 py-1.5 font-mono text-[10px] uppercase tracking-wider rounded-md border`.

## Interaction Specifications

### Tab Switching (explorar)

- URL updates to `?tab=rankings` on RANKINGS tab selection (matches existing `?tab=records` pattern)
- Tab content renders immediately (client-side state, no server round-trip for tab switch)
- Genre filter state resets when switching away from RANKINGS tab

### Genre Filter

- Clicking a genre chip fetches genre-specific leaderboard data
- Active genre chip gets highlighted state (secondary color)
- Only one genre active at a time (radio behavior, not checkbox)
- "GLOBAL" is always the first chip and is selected by default
- Horizontal scroll on mobile with `-webkit-overflow-scrolling: touch`

### Leaderboard Loading

- Initial load: Show 3 skeleton rows (48px height each, matching LeaderboardRow height)
- Skeleton: `bg-surface-container-low rounded-lg h-12 skeleton-shimmer` (existing pattern)
- Page size: 50 rows per load
- Pagination: Offset-based with "Load More" button at bottom (simplest approach, consistent with Phase 6 browse grid)
- "Load More" button: `font-mono text-[10px] uppercase tracking-wider text-on-surface-variant hover:text-on-surface`

### Own-User Highlight on Leaderboard

- If the authenticated user appears in the visible leaderboard rows, their row gets:
  - Left border: `border-l-2 border-primary`
  - Background: `bg-surface-container-high/50`
  - Rank number: `text-primary font-bold` instead of default

### Badge Award Notification

- When a badge is earned (from server action), a notification is created (type: `new_badge`)
- The notification appears in the existing NotificationBell component
- No toast/popup on the page — notification bell is sufficient (consistent with existing wantlist match pattern)

### Empty States

- Leaderboard with no data: Show empty state message (see Copywriting Contract)
- Profile with no ranking: Show "Vinyl Rookie" as default title, rank "--", score "0.0"
- Profile with no badges: BadgeRow renders nothing (no empty state message needed — badges appear as earned)

## State Specifications

### Data States for Leaderboard

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | Initial fetch in progress | 3 skeleton rows, genre filter chips visible but disabled |
| Populated | Data returned, rows > 0 | LeaderboardRow list with own-user highlight |
| Empty | Data returned, rows = 0 | Empty state message (see copywriting) |
| Error | Fetch failed | "Failed to load rankings. Try refreshing the page." in font-mono text-[10px] text-destructive |
| Loading more | Load More clicked, next page fetching | "Load More" button shows "LOADING..." text, disabled state |

### Data States for Profile Rank

| State | Condition | Visual |
|-------|-----------|--------|
| Ranked | `userRankings` row exists | Full RankCard with rank, title, rarity score, contribution score |
| Unranked | No `userRankings` row (new user, cron hasn't run) | RankCard with title "Vinyl Rookie", rank "--", scores "0.0" |

### Data States for Badges

| State | Condition | Visual |
|-------|-----------|--------|
| Has badges | `userBadges` query returns > 0 results | BadgeRow with bracket-style badge tags |
| No badges | `userBadges` query returns 0 results | BadgeRow renders nothing (no DOM output) |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab role semantics | RANKINGS tab button gets `role="tab"`, `aria-selected`, panel gets `role="tabpanel"` (matches existing pattern) |
| Leaderboard as list | Wrap rows in `<ol>` with `role="list"`, each row is `<li>` |
| Own-rank identification | Own row gets `aria-current="true"` for screen readers |
| Genre filter | Genre chip group uses `role="radiogroup"`, each chip `role="radio"` with `aria-checked` |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Ghost Protocol palette verified in Phase 1 |
| Reduced motion | Skeleton shimmer animation respects `prefers-reduced-motion` (existing global rule) |
| Badge semantics | Badge row uses `aria-label="Earned badges"`, each badge tag is a `<span>` with `title` attribute containing description |
