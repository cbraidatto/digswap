## Spacing Scale

Declared values (must be multiples of 4). Already defined in `globals.css` `@theme inline` block:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Gap between tab icon and label |
| sm | 8px | Internal padding of bottom bar items, avatar dropdown item padding |
| md | 16px | Horizontal padding of header and bottom bar, content area gutters |
| lg | 24px | Vertical padding around empty state content blocks |
| xl | 32px | Gap between empty state icon and heading |
| 2xl | 48px | Top margin of empty state container from content area top |
| 3xl | 64px | Reserved (not used in this phase) |

Exceptions: Bottom bar total height is 64px (3xl) including safe-area inset. Header height is 56px (not in the scale -- derived from 48px content + 8px vertical padding). Mobile touch targets for tab items are minimum 44px wide x 48px tall (accessibility minimum).

**Source:** `src/app/globals.css` lines 41-48, default 8-point scale

Accent is NOT used on: inactive tabs, body text, borders, dropdown items, backgrounds.

## Layout Contract

### Overall Structure

```
+-------------------------------------------+
|  AppHeader (fixed top, h-56px)            |
|  [VinylDig wordmark]        [Avatar ▾]   |
+-------------------------------------------+
|                                           |
|  Content Area (scrollable)                |
|  padding: 16px horizontal                 |
|  min-height: calc(100vh - 56px - 64px)    |
|                                           |
+-------------------------------------------+
|  BottomBar (fixed bottom, h-64px)         |
|  [Feed] [Perfil] [Explorar] [Comunidade] |
+-------------------------------------------+
```

### Header Specification

- Position: `fixed top-0 left-0 right-0`
- Height: 56px (48px content + 4px top padding + 4px bottom padding)
- Background: `oklch(0.18 0.02 55)` (secondary/surface)
- Bottom border: 1px solid `oklch(0.25 0.02 55)` (border)
- Backdrop filter: `blur(12px)` with background opacity at 0.85 for scroll-through effect
- Z-index: 40 (below grain overlay at 50)
- Left: VinylDig wordmark in Fraunces 20px semibold, color `oklch(0.72 0.14 65)` (accent)
- Right: User avatar (32px diameter), clickable to open dropdown
- Horizontal padding: 16px (md)

### Bottom Bar Specification

- Position: `fixed bottom-0 left-0 right-0`
- Height: 64px + `env(safe-area-inset-bottom)` for iOS home bar
- Background: `oklch(0.18 0.02 55)` (secondary/surface)
- Top border: 1px solid `oklch(0.25 0.02 55)` (border)
- Backdrop filter: `blur(12px)` with background opacity at 0.85
- Z-index: 40
- Layout: 4 equal-width flex items, centered content
- Padding bottom: `env(safe-area-inset-bottom, 0px)`

### Bottom Bar Item Specification

- Layout: flex column, items-center, gap 4px (xs)
- Icon size: 24px x 24px
- Label: DM Sans 12px semibold
- Touch target: minimum 44px wide x 48px tall (entire item area)
- **Active state:** Icon + label in accent color `oklch(0.72 0.14 65)`, no pill/capsule/underline
- **Inactive state:** Icon + label in muted-foreground `oklch(0.60 0.03 60)`
- Transition: `color 150ms ease-in-out`

### Tab Icons (Lucide)

| Tab | Icon name | Lucide component |
|-----|-----------|-----------------|
| Feed | `Disc3` | Spinning disc/record icon |
| Perfil | `User` | Person silhouette |
| Explorar | `Search` | Magnifying glass |
| Comunidade | `Users` | People group |

**Source:** 02-CONTEXT.md D-12 (Lucide where possible), Claude's discretion for exact choices

### Content Area Specification

- Margin top: 56px (header height)
- Margin bottom: 64px (bottom bar height)
- Padding: 16px horizontal (md)
- Overflow-y: auto
- Background: transparent (inherits page background)

## Interaction States

### Bottom Bar Tab Tap

| State | Visual |
|-------|--------|
| Default (inactive) | Icon + label in muted-foreground |
| Active (current tab) | Icon + label in accent color |
| Pressed | No explicit pressed state (instant color transition) |
| Focus-visible | Standard ring outline (`--ring`) for keyboard navigation |

### Avatar Dropdown

| State | Visual |
|-------|--------|
| Closed | Avatar image or initials circle |
| Open | Radix dropdown appears below-right with fade-in (100ms) |
| Item hover | Background changes to secondary color |
| Item focus-visible | Standard ring outline for keyboard navigation |

### Navigation Transitions

- Tab switching: No page transition animation. Instant rendering via Next.js route change.
- Within-tab navigation: Standard Next.js link behavior, no custom transitions.
- Deep link load: Correct tab highlighted on initial page load via `usePathname()` matching.

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Bottom bar semantics | `<nav>` element with `aria-label="Main navigation"` |
| Active tab | `aria-current="page"` on active tab link |
| Tab links | `<a>` tags (Next.js `Link`) with descriptive text (icon is decorative, label is the accessible name) |
| Avatar dropdown | Radix DropdownMenu handles ARIA roles automatically |
| Keyboard navigation | Tab key moves through bottom bar items and header; Enter/Space activates |
| Reduced motion | Respect `prefers-reduced-motion` (already in globals.css) -- disable backdrop-filter animations |
| Color contrast | Accent on surface: OKLCH 0.72 on 0.18 = ratio > 4.5:1 (WCAG AA). Muted-foreground on background: OKLCH 0.60 on 0.13 = ratio > 4.5:1 (WCAG AA) |
| Touch targets | Minimum 44px x 48px per tab item |
| Focus ring | Uses `--ring` variable (accent color) via shadcn defaults |

## Routing Contract

| Route | Tab | Description |
|-------|-----|-------------|
| `/feed` | Feed | Empty state placeholder |
| `/feed/*` | Feed | Future sub-routes highlight Feed tab |
| `/perfil` | Perfil | Current user profile placeholder |
| `/perfil/*` | Perfil | Future sub-routes highlight Perfil tab |
| `/explorar` | Explorar | Empty state placeholder |
| `/explorar/*` | Explorar | Future sub-routes highlight Explorar tab |
| `/comunidade` | Comunidade | Empty state placeholder |
| `/comunidade/*` | Comunidade | Future sub-routes highlight Comunidade tab |
| `/settings` | None | Settings page (no tab highlighted, accessed via dropdown) |
| `/settings/*` | None | Settings sub-routes |

**Active tab detection:** `usePathname()` checks if current path starts with the tab's base path. E.g., pathname starts with `/feed` -> Feed tab is active.

**Route groups under `(protected)/`:**
- `(protected)/(feed)/feed/` -- Feed tab routes
- `(protected)/(profile)/perfil/` -- Perfil tab routes
- `(protected)/(explore)/explorar/` -- Explorar tab routes
- `(protected)/(community)/comunidade/` -- Comunidade tab routes

**Default redirect:** After onboarding completes, redirect to `/feed`.
