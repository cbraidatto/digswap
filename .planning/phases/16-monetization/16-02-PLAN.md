# 16-02 — Entitlement layer: quota gate + tier helpers

**Owner**: Codex
**Phase**: 16 — Monetization
**Depends on**: 16-01 (subscriptions table populated, service-role writes working)

## Goal

A single authoritative module that any server action can call to check if a user can initiate a trade and to increment their counter. No entitlement logic scattered across unrelated files.

## Context

- Free tier: max 5 trades per calendar month (`tradesThisMonth >= 5` → blocked)
- Premium: no limit
- `tradesMonthReset` tracks when the month window started — reset counter when `now() > tradesMonthReset + 1 month`
- Counter increment happens at trade request creation time (Phase 9/14 server action `createTradeRequest`)
- `profiles.subscriptionTier` is a denormalized fast-read copy — use for display only, not for quota decisions (use `subscriptions.plan` as source of truth)

## Tasks

### Task 1 — Entitlement helpers module

Create `apps/web/src/lib/entitlements.ts`:

```ts
export const FREE_TRADE_LIMIT = 5;

export type SubscriptionPlan = "free" | "premium_monthly" | "premium_annual";

export function isPremium(plan: SubscriptionPlan): boolean {
  return plan === "premium_monthly" || plan === "premium_annual";
}

// Server-side: read subscription for current user
export async function getUserSubscription(userId: string): Promise<{
  plan: SubscriptionPlan;
  tradesThisMonth: number;
  tradesMonthReset: Date | null;
} | null>

// Returns null if no subscription row (treat as free, 0 trades)
// Resets counter if tradesMonthReset is more than 1 month ago (lazy reset)

export async function canInitiateTrade(userId: string): Promise<{
  allowed: boolean;
  reason?: "no_limit" | "under_limit" | "limit_reached";
  tradesUsed: number;
  tradesLimit: number | null; // null = unlimited
}>

// Increments tradesThisMonth for free users (no-op for premium)
// Also updates tradesMonthReset if this is the first trade of the month
export async function incrementTradeCount(userId: string): Promise<void>
```

### Task 2 — Wire into createTradeRequest server action

- In `apps/web/src/actions/trades.ts` (or wherever `createTradeRequest` lives), call `canInitiateTrade` before inserting:
  ```ts
  const check = await canInitiateTrade(user.id);
  if (!check.allowed) {
    return { error: "TRADE_LIMIT_REACHED", tradesUsed: check.tradesUsed };
  }
  // ... insert trade
  await incrementTradeCount(user.id);
  ```
- Return typed error `TRADE_LIMIT_REACHED` (not a thrown exception) so the client can show an upgrade modal
- `incrementTradeCount` called AFTER successful insert (not before — don't count failed trade requests)

### Task 3 — Lazy month reset logic

In `getUserSubscription`, before returning:
- If `tradesMonthReset` is null OR `now() > tradesMonthReset + 30 days`:
  - Reset `tradesThisMonth = 0`, `tradesMonthReset = now()` via service-role client
  - Return the reset values

This avoids needing a cron job for counter resets while keeping logic simple.

### Task 4 — `getQuotaStatus` helper for UI components

```ts
// Convenience wrapper for display — returns everything the quota banner needs
export async function getQuotaStatus(userId: string): Promise<{
  plan: SubscriptionPlan;
  isPremium: boolean;
  tradesUsed: number;
  tradesLimit: number | null;
  percentUsed: number | null; // 0-100, null if unlimited
}>
```

Used in server components that render the quota banner on `/trades`.

## Verification

- [ ] Free user with 0 trades → `canInitiateTrade` returns `allowed: true`
- [ ] Free user with 5 trades → `canInitiateTrade` returns `allowed: false, reason: "limit_reached"`
- [ ] Premium user with 100 trades → `canInitiateTrade` returns `allowed: true, reason: "no_limit"`
- [ ] Month reset: user with `tradesMonthReset` 31+ days ago → counter resets to 0 on next call
- [ ] `incrementTradeCount` on premium user → no-op (count unchanged)
- [ ] `createTradeRequest` with limit-reached user → returns `TRADE_LIMIT_REACHED` error
