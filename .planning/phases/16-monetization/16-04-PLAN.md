# 16-04 ŌĆö Premium surface gates: quota banner, trade block, premium badges

**Owner**: Claude
**Phase**: 16 ŌĆö Monetization
**Depends on**: 16-02 (entitlement helpers), 16-03 (pricing page exists for upgrade CTA)

## Goal

Integrate quota enforcement into the trades inbox and profile surfaces. Free users see their counter and hit a hard block when they reach 5/month. Premium users see no friction.

## Tasks

### Task 1 ŌĆö Quota banner on `/trades` inbox

File: `apps/web/src/app/(protected)/trades/page.tsx`

At the top of the trades list, render a `<TradeQuotaBanner>` server component:

```tsx
// Server component ŌĆö fetches quota server-side
const quota = await getQuotaStatus(user.id);
// Pass to banner
```

**`TradeQuotaBanner` component** (`apps/web/src/components/trades/TradeQuotaBanner.tsx`):

Props: `{ plan, tradesUsed, tradesLimit, percentUsed }`

Renders:
- If premium: nothing (return null)
- If free + under limit (< 5): compact info strip
  - `bg-[#111008] border border-[#2a2218]` bar
  - Text: `TRADES THIS MONTH  X / 5` in mono
  - Progress bar: `bg-[#c8914a]` fill, `bg-[#1a1208]` track
  - Subtle "Upgrade" link ŌåÆ `/pricing`
- If free + at limit (>= 5): warning banner
  - `bg-[#1a0a00] border border-[#c8914a]/40` with amber tint
  - Text: `TRADE LIMIT REACHED ŌĆö 5 / 5 trades used this month`
  - CTA button: `UPGRADE_TO_PREMIUM` ŌåÆ `/pricing`

### Task 2 ŌĆö Trade initiation block modal

When `createTradeRequest` returns `TRADE_LIMIT_REACHED` error (from 16-02 wiring), the client must show an upgrade modal instead of the generic error toast.

File: `apps/web/src/components/trades/TradeLimitModal.tsx` (client component):

```tsx
interface Props {
  open: boolean;
  onClose: () => void;
  tradesUsed: number;
}
```

**Visual**:
- Modal overlay: `bg-black/60 backdrop-blur-sm`
- Modal card: `bg-[#111008] border border-[#c8914a]/30` max-w-sm centered
- Header: vinyl record icon + `MONTHLY LIMIT REACHED`
- Body: "You've used all 5 trades this month. Upgrade to Premium for unlimited trades."
- Primary CTA: `UPGRADE_TO_PREMIUM` ŌåÆ `createPortalSession` or `/pricing`
- Secondary: `CLOSE` ŌĆö dismiss modal
- Resets on month rollover (handled server-side)

Wire into the component that calls `createTradeRequest` ŌĆö check returned error and trigger modal.

### Task 4 ŌĆö `/trades/new` or initiate-trade flow guard

Wherever the user triggers a new trade (propose trade button on a record or profile):
- Before showing the trade form, call `canInitiateTrade` server-side
- If `allowed: false`, render the `TradeLimitModal` immediately (skip the form)
- Or redirect to `/pricing?reason=limit`

This is a server-side check (not just client) ŌĆö important for security.
