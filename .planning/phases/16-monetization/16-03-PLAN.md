# 16-03 — Pricing page + subscription management UI

**Owner**: Claude
**Phase**: 16 — Monetization
**Depends on**: 16-01 (checkout/portal actions available)

## Goal

A `/pricing` page where users can compare free vs premium and subscribe, plus a `/settings/billing` page to manage their current subscription.

## Context

- Design system: dark warm palette (`#0d0d0d` bg, `#e8dcc8` text, `#c8914a` amber accent), `font-mono` for data
- CTA buttons call `createCheckoutSession` (client) → redirect to Stripe Checkout
- "Manage subscription" calls `createPortalSession` → redirect to Stripe Customer Portal
- Must show current plan state (free or active premium) — read via `getQuotaStatus` server component

## Tasks

### Task 1 — `/pricing` page

File: `apps/web/src/app/(public)/pricing/page.tsx` (public route, no auth required to view)

Layout: two plan cards side by side (Free | Premium), toggle for Monthly/Annual pricing.

**Free plan card**:
- Price: $0/month
- Features: 5 trades/month, full collection import, wantlist matching, community access
- CTA: "Get started free" → `/signup` (or "Your current plan" if already signed in on free)

**Premium plan card** (highlighted — amber border):
- Monthly: $X/month | Annual: $Y/year (save ~17%)
- Features: Unlimited trades, collection analytics, premium groups, priority matching, supporter badge
- CTA: "Upgrade to Premium" → calls `createCheckoutSession(priceId)` server action
- If already premium: "Manage subscription" → calls `createPortalSession()`

Price toggle (Monthly / Annual):
- Client component with `useState("monthly")`
- Switches displayed price and the `priceId` passed to checkout action

**Visual treatment**:
- Section header: `PRICING` in small-caps mono tracking-widest above `Choose your dig level`
- Free card: `bg-[#111008] border border-[#2a2218]`
- Premium card: `bg-[#1a1208] border border-[#c8914a]/40` with amber glow shadow
- Feature list uses `✓` checkmarks in `text-[#c8914a]`
- "Most popular" chip on Premium: `bg-[#c8914a] text-[#0d0d0d] font-mono text-[10px] uppercase px-2 py-0.5`

### Task 2 — `/settings/billing` page

File: `apps/web/src/app/(protected)/settings/billing/page.tsx`

Server component that reads `getQuotaStatus(user.id)`:

**Layout**:
- Header: `BILLING` label + current plan pill (`FREE` or `PREMIUM` in amber)
- If free: quota row — "Trades this month: X / 5" with progress bar
- If premium: shows next renewal date from `currentPeriodEnd`
- "Upgrade" button (free) → `createCheckoutSession` CTA
- "Manage subscription" button (premium) → `createPortalSession` CTA — opens Stripe portal for cancel/change
- Success message when redirected back with `?success=true` query param

**Success state** (when `?success=true` in URL):
- Show `bg-[#0a1a0a] border border-[#7ac87a]/20` card: "Subscription activated — welcome to Premium"
- Auto-disappears after 5s or on dismiss
