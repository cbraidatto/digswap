# 16-01 — Stripe webhook handler + checkout/portal server actions

**Owner**: Codex
**Phase**: 16 — Monetization
**Depends on**: subscriptions schema (already in Drizzle), Stripe account configured

## Goal

Wire up Stripe so subscription events from Stripe's servers sync into the `subscriptions` table, and give users a way to start/manage their subscription from the web app.

## Context

- `subscriptions` table already exists in Drizzle schema (`apps/web/src/lib/db/schema/subscriptions.ts`)
- RLS: user selects own, only service role can insert/update → webhook handler must use service-role client
- `profiles.subscriptionTier` is the denormalized fast-read field (kept in sync by webhook)
- Free tier: `plan = "free"`, `tradesThisMonth` counter, `tradesMonthReset` date
- Premium tiers: `plan = "premium_monthly"` | `"premium_annual"`

## Tasks

### Task 1 — Install + configure Stripe SDK

- Add `stripe` package: `pnpm --filter web add stripe`
- Add env vars to `.env.local` (and document in `.env.example`):
  - `STRIPE_SECRET_KEY` — server-only, never exposed to client
  - `STRIPE_WEBHOOK_SECRET` — for signature verification
  - `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` — safe for client
  - `NEXT_PUBLIC_STRIPE_PRICE_MONTHLY` — Stripe Price ID for monthly plan
  - `NEXT_PUBLIC_STRIPE_PRICE_ANNUAL` — Stripe Price ID for annual plan
- Create `apps/web/src/lib/stripe.ts`:
  ```ts
  import Stripe from "stripe";
  export const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
    apiVersion: "2024-06-20",
    typescript: true,
  });
  ```

### Task 2 — Stripe webhook endpoint

Create `apps/web/src/app/api/stripe/webhook/route.ts`:

- Verify signature with `stripe.webhooks.constructEvent(body, sig, STRIPE_WEBHOOK_SECRET)`
- Return 400 on signature failure (never expose error details)
- Handle events:
  - `checkout.session.completed` → upsert subscription row, set `plan`, `stripeCustomerId`, `stripeSubscriptionId`, `status = "active"`, `currentPeriodStart/End`; also UPDATE `profiles.subscription_tier`
  - `customer.subscription.updated` → update `status`, `plan`, `currentPeriodStart/End`
  - `customer.subscription.deleted` → set `plan = "free"`, `status = "cancelled"`, clear period dates; revert `profiles.subscription_tier = "free"`
  - `invoice.payment_failed` → set `status = "past_due"`
- All DB writes use the service-role Supabase client (bypasses RLS)
- Return `{ received: true }` with 200 on success
- Disable Next.js body parsing: `export const config = { api: { bodyParser: false } }` — use `request.text()` for raw body
- Must NOT be behind auth middleware — add `/api/stripe/webhook` to middleware public routes
