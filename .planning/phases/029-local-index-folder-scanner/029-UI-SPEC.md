## Interaction States

### Screen State Machine

```
EMPTY → (user clicks "Escolher Pasta") → SCANNING
SCANNING → (scan completes, no errors) → LIBRARY
SCANNING → (scan completes, has errors) → ERROR_SUMMARY → (dismiss) → LIBRARY
LIBRARY → (user clicks "Re-scan") → SCANNING
LIBRARY → (user clicks "Re-scan Completo") → SCANNING
```

### View Toggle

- Default view on first visit: List mode
- Toggle persists across screen transitions (stored in component state)
- Toggle does NOT persist across app restarts (no persistence needed this phase)

### Progress Bar

- Determinate: width derived from `filesProcessed / filesFound`
- Updates throttled at 50ms from main process (per D-03)
- File ticker shows the current file path being processed, truncated from the left
- On completion, progress bar reaches 100% then fades out as results appear

### Folder Picker

- Native OS dialog via `dialog.showOpenDialog({ properties: ['openDirectory'] })`
- Dialog title: "Selecionar pasta da biblioteca"
- After selection, scan starts immediately (no confirmation step)
- If user cancels dialog, return to previous state (empty or library)
