## Objective

Refactor the Electron shell into two explicit surfaces:

1. A remote main window that loads the real DigSwap web app.
2. A local trade window that keeps the privileged trade runtime.

This is a shell/security refactor only. It must not change the existing trade runtime contract.

## Context

Today the desktop app is a standalone local mini-app. The target architecture is closer to Discord/Notion:

- Main window: the real web product in a native shell.
- Trade window: a local privileged renderer for lobby, transfer, and completion.

The current preload in `apps/desktop/src/preload/index.ts` exposes the full desktop bridge. That is acceptable for a local renderer, but not acceptable once the main window loads remote web content.

The existing hidden WebRTC bridge window in `peer-session.ts` stays untouched in this plan.

## Interfaces

### Remote main window preload

Add a narrow bridge in `apps/desktop/src/shared/ipc-types.ts`:

```typescript
export interface DesktopMainShellBridge {
  isDesktop(): boolean;
  getAppVersion(): Promise<string>;
}
```

Expose it from `src/preload/main.ts` on a separate namespace:

```
window.desktopShell
```

The remote web app must not receive `window.desktopBridge`.

### Trade window preload

Move the current privileged preload into `src/preload/trade.ts` and keep:

- getTradeDetail
- startTransfer
- cancelTransfer
- confirmCompletion
- onLobbyStateChanged
- onTransferProgress
- onTransferComplete
- existing auth/settings/runtime methods needed by the local trade renderer

## Tasks

### Task 1 - Split window creation and secure the remote main window

Refactor `apps/desktop/src/main/window.ts` to export:

- `createMainWindow(config)`
- `createTradeWindow()`
- `getMainWindow()`
- `getTradeWindow()`
- `focusMainWindow()`
- `focusTradeWindow()`

Requirements:

- Main window loads `config.siteUrl` directly. In dev that resolves to `http://localhost:3000`.
- Trade window loads a local renderer HTML entry from the Electron bundle.
- Main window:
  - `contextIsolation: true`
  - `sandbox: true`
  - `nodeIntegration: false`
  - popup opens external only
  - navigation blocked unless origin is the configured DigSwap site origin
  - permission requests denied by default
- Trade window:
  - keeps the local privileged preload
  - does not load any remote URL

### Task 2 - Split the preload layer

Update `apps/desktop/electron.vite.config.ts` so Electron builds two preload entries:

- `src/preload/main.ts`
- `src/preload/trade.ts`

Move the current preload implementation:

- minimal main-shell bridge → `main.ts`
- privileged trade bridge → `trade.ts`

Keep the contract names distinct:

- `window.desktopShell` for the remote web app
- `window.desktopBridge` for the local trade renderer

### Task 3 - Route protocol and IPC events by window role

Refactor `apps/desktop/src/main/index.ts` and `apps/desktop/src/main/ipc.ts` so that:

- `digswap://trade/...` opens or focuses the trade window
- `digswap://auth/callback` stays in the main process auth runtime
- session/app-level events may go to both windows if needed
- `desktop:lobby-state-changed`, `desktop:transfer-progress`, `desktop:transfer-complete` go only to the trade window

Do not change the shape of the trade runtime IPC surface in this plan.

- remote main window does not have privileged trade/file APIs

## Success Criteria

- Main window is remote and locked down
- Trade window is local and privileged
- Preload surfaces are split cleanly by trust boundary
- No privileged trade runtime API is reachable from remote content
- Existing handoff and trade runtime flow still works after the shell split
