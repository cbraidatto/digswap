## Objective

Establish a safe explicit auth/session contract between the remote main window and the Electron main process, so the trade runtime can use the same authenticated user without scraping browser storage.

## Context

After 18-01 and 18-02:

- main window = remote DigSwap web app
- trade window = local privileged trade renderer

That creates one remaining gap: the trade runtime in main process still needs a valid Supabase session to consume handoff tokens, acquire leases, and reconcile receipts.

The correct fix is explicit session sync:
- the web app knows when auth changes
- the web app sends tokens to the desktop main process through a minimal preload bridge
- the main process imports and persists that session in the existing safeStorage-backed vault

## Interfaces

Extend the minimal main-shell bridge in `apps/desktop/src/shared/ipc-types.ts`:

```typescript
export interface DesktopShellSessionPayload {
  accessToken: string;
  refreshToken: string;
}

export interface DesktopMainShellBridge {
  isDesktop(): boolean;
  getAppVersion(): Promise<string>;
  syncSession(session: DesktopShellSessionPayload | null): Promise<void>;
}
```

Main window namespace remains:

```
window.desktopShell
```

The trade renderer keeps using `window.desktopBridge` unchanged.

## Tasks

### Task 1 - Add explicit session import on the desktop side

Refactor `apps/desktop/src/main/supabase-auth.ts` to expose:

```typescript
importSession(tokens: { accessToken: string; refreshToken: string }): Promise<void>;
clearImportedSession(): Promise<void>;
```

Implementation guidance:

- use the existing Supabase client in main process
- call `auth.setSession(...)`
- let the existing storage adapter persist the session into the encrypted vault
- preserve current external-browser auth flow as a valid fallback

Wire IPC in `apps/desktop/src/main/ipc.ts` for `desktop-shell:sync-session` and expose it only through `src/preload/main.ts`.

### Task 2 - Add a tiny web adapter that mirrors auth state into the desktop shell

Create a client component: `apps/web/src/components/desktop/desktop-session-sync.tsx`

Behavior:

- detect `window.desktopShell`
- subscribe to browser-side Supabase auth state
- on login/session refresh: call `window.desktopShell.syncSession({ accessToken, refreshToken })`
- on logout: call `window.desktopShell.syncSession(null)`

Mount it high enough in `apps/web/src/app/layout.tsx` that both auth and protected routes sync correctly.

This component must be tiny and side-effect-only. It must not own UI.

### Task 3 - Verify handoff/runtime works with synced auth

Keep the current `/desktop/open` and `digswap://trade` flow unchanged.

Verify that:
- user logs in on the remote web app
- desktop main process receives synced session
- trade handoff opens trade window without requiring a second local login
- logout in the web app clears the desktop session used by the runtime

Do not add any filesystem or transfer methods to `window.desktopShell`.
