## Objective

Create a dedicated local trade renderer that becomes the content of the new trade window. This renderer replaces the current all-purpose local mini-app and focuses only on the trade runtime flow.

## Context

After 18-01, the main desktop window is the real DigSwap web app. The local renderer is no longer the shell of the entire product.

What remains local is only the privileged trade flow:
- lobby
- transfer
- completion

Existing screens in `apps/desktop/src/renderer/src/` already contain most of this UI. The result must be a dedicated trade-only entry point under `renderer-trade`.

## Interfaces

The renderer contract stays on `window.desktopBridge` and remains preload-only.

Expected runtime methods:

```typescript
getBootstrapState(): Promise<DesktopBootstrapState>;
getTradeDetail(tradeId: string): Promise<TradeDetail>;
startTransfer(tradeId: string): Promise<void>;
cancelTransfer(tradeId: string): Promise<void>;
confirmCompletion(tradeId: string, rating: number): Promise<void>;
openFileInExplorer(filePath: string): Promise<void>;
onProtocolPayload(cb): () => void;
onLobbyStateChanged(cb): () => void;
onTransferProgress(cb): () => void;
onTransferComplete(cb): () => void;
```

Do not change event shapes in this plan.

## Tasks

### Task 1 - Create the dedicated trade renderer entry point

Add a new local renderer entry under `apps/desktop/src/renderer-trade/` with:

- `index.html`
- `src/main.tsx`
- `src/app.tsx`
- trade-specific styles

This renderer becomes the only content for the trade window created in 18-01.

### Task 2 - Move or rewrite the trade flow UI

Implement the local trade app with exactly three states:

1. Lobby
2. Transfer
3. Completion

Behavior:

- read `lastProtocolPayload` and/or active handoff on bootstrap
- if a trade-handoff payload is present and session is already available, open the requested trade automatically
- subscribe to lobby/progress/complete events
- keep cancellation and completion flows intact

### Task 3 - Retire the old local shell from the primary path

The old local screens (Login, Inbox, Settings) must no longer be the default desktop experience.

They may remain temporarily in the repo during transition, but they must not be the content of the main window or the trade window after this plan.
