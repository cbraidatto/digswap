# Phase 6 — UI Design Contract

> Visual and interaction contract for Discovery + Notifications. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | base-nova (neutral base, CSS variables, RSC) |
| Component library | base-ui (via shadcn base-nova style) |
| Icon library | lucide-react + Material Symbols Outlined |
| Font (sans) | Inter (400, 500, 600) |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#10141a` (--background/--surface) | Page background, main content area |
| Secondary (30%) | `#181c22` (--card/--surface-container-low) | Cards, notification rows, record cards, filter area |

| Destructive | `#ffb4ab` (--destructive) | Error states only |

**Accent (`#6fdd78` / --primary) reserved for:**
- Active tab indicator (bottom border)
- Search input cursor blink
- Search input left border (4px)
- Terminal prompt character `>`
- Unread notification count badge background
- "Mark all read" button text
- Active filter chip background (at 10% opacity: `bg-primary/10 text-primary border-primary`)
- Record card hover title color
- Result count number in search results header

**Secondary accent (`#aac7ff` / --secondary) reserved for:**
- "Rare" rarity badge
- "View all" link in notification dropdown

**Tertiary (`#ffb689` / --tertiary) reserved for:**
- "Ultra Rare" rarity badge
- Wantlist match notification icon tint

**Notification-specific colors:**
- Unread badge: `bg-destructive text-destructive-foreground` (red dot with white number, following standard bell badge convention -- destructive color draws attention)
- Unread notification row: `bg-surface-container-high` (elevated from default row background)
- Read notification row: `bg-surface-container-low` (same as default card background)
- Notification type icons: `text-primary` for wantlist_match, `text-on-surface-variant` for all placeholder types

**Filter chip states:**
- Inactive: `bg-surface-container-low text-on-surface-variant border border-outline-variant/20`
- Active: `bg-primary/10 text-primary border border-primary`
- Hover (inactive): `bg-surface-container text-on-surface border border-outline-variant/40`

## Interaction Contracts

### Search Input (Record Search)

| Trigger | Behavior |
|---------|----------|
| Type >= 2 chars | Debounce 300ms, then fire server action `searchRecords` |
| Type < 2 chars | Clear results, hide results area |
| Clear input | Clear results, hide results area |
| Server action pending | Show `searching...` below input (same pattern as Phase 5) |
| Results arrive (>0) | Show result count header + record cards with owners |
| Results arrive (0) | Show `no records found matching "{query}"` |

### Tab Switching (DIGGERS / RECORDS)

| Trigger | Behavior |
|---------|----------|
| Click inactive tab | Switch tab content, preserve search state in each tab independently |
| Page load | Default to DIGGERS tab (preserves Phase 5 behavior) |
| URL param (optional) | `?tab=records` to deep-link to RECORDS tab |

### Filter Chips (Genre/Decade)

| Trigger | Behavior |
|---------|----------|
| Click inactive chip | Activate chip, deactivate previous in same row, trigger browse query |
| Click active chip | Deactivate chip (clear that filter dimension), re-query |
| Both genre + decade active | AND filter: results match both genre AND decade range |
| Neither active | No browse results shown (initial state) -- search results only |

### Notification Bell

| Trigger | Behavior |
|---------|----------|
| New notification (Realtime INSERT) | Increment badge count, prepend to recent list (max 5) |
| Click bell (closed) | Open Popover dropdown, show 5 most recent notifications |
| Click bell (open) | Close Popover dropdown |
| Click outside dropdown | Close Popover dropdown |
| Click notification row | Navigate to `notification.link`, mark as read, close dropdown |
| Click "Mark all read" | Mark all notifications as read via server action, reset badge to 0 |
| Click "View all" | Navigate to `/notifications` page |
| Badge display | Number 1-9 shown; 10+ shows "9+" |
| Zero unread | No badge visible (not "0") |

### Notification Preferences Toggles

| Trigger | Behavior |
|---------|----------|
| Toggle checkbox | Optimistic UI update, fire server action `updatePreferences` |
| Server action fails | Revert toggle, show toast error |
| First settings visit | Lazy-create preferences row with all defaults (true for in-app, true for email where applicable) |
| Disabled toggle click | No-op (pointer-events-none via disabled attribute) |

### Wantlist Match (Background)

| Trigger | Behavior |
|---------|----------|
| User adds record to collection | Check wantlist_items for matching release_id (excluding adder). For each match: insert notification + send email if enabled. |
| Import pipeline completes | Batch check all newly imported release IDs against wantlist_items. Insert notifications + emails. |
| Email preference disabled | Skip email send, still create in-app notification |
| Release has no discogs_release_id | Skip matching entirely (unreliable without canonical ID) |

## Loading & Error States

### Loading States

| Surface | Loading Pattern |
|---------|----------------|
| Record search results | `searching...` text below input (JetBrains Mono, 10px, muted) |
| Browse grid | 4x skeleton cards (reuse skeleton-shimmer class) |
| Notification bell initial load | No loading state -- badge shows 0 until count fetched |
| Notification dropdown | 3x skeleton rows (height 64px each) |
| Notifications page | 5x skeleton rows (full width) |
| Suggested section | 4x skeleton cards |
| Settings preferences | Inline skeleton for toggle rows until preferences loaded |

### Error States

| Surface | Error Pattern |
|---------|---------------|
| Record search failure | `Search failed. Please try again.` below input, mono 12px, destructive color |
| Browse query failure | `Failed to load records. Please try again.` in grid area |
| Notification dropdown load failure | `Could not load notifications.` single row in dropdown |
| Email send failure | Silent (console.error only -- never blocks user flow) |
| Preferences save failure | Toast: `Failed to update preferences. Please try again.` |
| Realtime subscription failure | Silent reconnect (Supabase client handles reconnection internally) |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Bell button | `aria-label="Notifications"`, `aria-expanded` toggles with popover state |
| Unread badge | `aria-label="{N} unread notifications"` on badge element |
| Tab bar | `role="tablist"`, each tab `role="tab"` with `aria-selected`, tab panel `role="tabpanel"` |
| Search input | `role="searchbox"`, `aria-label="Search records by name or artist"` |
| Filter chips | Each chip is a `<button>` with `aria-pressed` reflecting active state |
| Notification rows | `role="listitem"` inside `role="list"` container |
| Disabled toggles | `disabled` attribute + `aria-disabled="true"` |
| Focus management | Bell popover: focus first notification on open. Tab: keyboard arrow keys not required (simple click/keyboard enter) |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 ratio). `#6fdd78` on `#10141a` = 9.7:1. `#becab9` on `#10141a` = 8.4:1. |
| Reduced motion | All animations respect `prefers-reduced-motion: reduce` (already in globals.css) |
