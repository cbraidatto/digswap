Create apps/desktop/src/main/audio/preview-uploader.ts:
- Export: `uploadPreviewToStorage(client: SupabaseClient, tradeId: string, userId: string, proposalItemId: string, localPath: string): Promise<string>`
- Read file into Buffer with fs/promises.readFile
- Derive storage path: `${tradeId}/${userId}/${proposalItemId}${path.extname(localPath)}`
- Call `client.storage.from('trade-previews').upload(storagePath, fileBuffer, { upsert: false, contentType: 'application/octet-stream' })`
- If Supabase returns an error, throw it. Return the storagePath string on success.
- Do NOT use upsert:true — first-write wins; duplicate upload attempts are a caller bug.

Create apps/desktop/src/main/audio/upload-pipeline.ts:
- Export `AudioPrepResult { sourcePath: string; sha256: string; specs: AudioSpecs; previewStoragePath: string; proposalItemId: string; }`
- Export `runAudioUploadPipeline(client: SupabaseClient, tradeId: string, userId: string, proposalItemId: string, sourcePath: string): Promise<AudioPrepResult>`
- Steps in order:
  1. `extractSpecs(sourcePath)` — throws AudioPipelineError.FILE_TOO_SHORT if < 120s (already guarded in 27-01)
  2. `computeFileSha256(sourcePath)` — compute hash of source (not preview)
  3. Write sha256 to DB: `client.from('trade_proposal_items').update({ file_hash: sha256 }).eq('id', proposalItemId)` — ONLY if file_hash IS NULL (to preserve immutability). Use: `.is('file_hash', null)`. Throw if error.
  4. `generatePreview(sourcePath, specs, os.tmpdir())` — cut to temp dir
  5. `uploadPreviewToStorage(client, tradeId, userId, proposalItemId, previewResult.previewPath)` — upload
  6. Delete local preview temp file with `fsp.unlink` (best-effort, swallow ENOENT)
  7. Return AudioPrepResult
- The SHA-256 immutability constraint: step 3 uses `.is('file_hash', null)` so a second call to this function for the same item is a no-op on the DB column (the update matches zero rows).
