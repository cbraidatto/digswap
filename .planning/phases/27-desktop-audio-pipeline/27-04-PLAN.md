Create apps/desktop/src/renderer/src/SpectralVisualizer.tsx.

Props: `{ audioUrl: string; width?: number; height?: number }`
Defaults: width=400, height=160

Implementation:
1. `useRef<HTMLCanvasElement>()` for canvas element
2. On `audioUrl` change: fetch the audio file as ArrayBuffer, decode via `new AudioContext().decodeAudioData()`, then run FFT analysis.
3. FFT analysis: create OfflineAudioContext (same sample rate, same length). Create AnalyserNode with `fftSize = 2048`. Pipe buffer through source → analyser. Use `analyser.getFloatFrequencyData()` sampled at 256 time slices across the buffer's duration to build a [timeSlices × freqBins] amplitude matrix.
4. Render the matrix to canvas using ImageData: map amplitude (dB, range -120 to 0) to a color gradient — black (#000000) at -120 dB to amber (#c8914a) at -40 dB to white (#ffffff) at 0 dB. Use `ctx.putImageData()`.
5. Draw frequency axis labels on left side (20Hz, 1kHz, 10kHz, 20kHz) in #4a4035, font "9px monospace".
6. Loading state: render a pulsing "Analyzing…" text centered on the canvas.
7. Error state: render "Analysis failed" in #4a4035.
8. Clean up AudioContext on unmount with `audioContext.close()`.

The component is purely visual — it does NOT play audio, only displays spectrum. This is a static analysis pass, not real-time.

Styling: rounded border border-[#2a2218], display block, bg-[#0d0d0d].

Create apps/desktop/src/renderer/src/AudioPrepScreen.tsx:

Props: `{ tradeId: string; proposalItems: Array<{ id: string; title: string; artist: string }>; onComplete: (results: AudioPrepResult[]) => void; onCancel: () => void; }`

State:
- `currentIndex: number` (starts at 0)
- `results: AudioPrepResult[]` (accumulates per-item results)
- `itemState: 'idle' | 'preparing' | 'done' | 'error'`
- `errorMessage: string | null`

Behavior:
1. Display list of all items with status indicator (pending / in-progress / done / error).
2. "Upload Audio" button for the current item. Button label: "Select & Upload {title}".
3. On click: set state to 'preparing', call `window.desktopBridge.selectAndPrepareAudio(tradeId, proposalItems[currentIndex].id)`.
4. On success: push result to results array, advance currentIndex. If all done, call `onComplete(results)`.
5. On error: show error message (distinguish AudioPipelineError codes: FILE_TOO_SHORT → "File must be at least 2 minutes long", FFMPEG_FAILED → "Could not read audio file — check format", network error → "Upload failed, check connection"). Allow retry.
6. On completion of each item, show `<SpectralVisualizer audioUrl={previewStoragePath} />` below the item — construct full Supabase Storage URL from `previewStoragePath`. The public URL pattern is: `{SUPABASE_URL}/storage/v1/object/public/trade-previews/{storagePath}`.
   The SUPABASE_URL is available from the same env vars the main process uses — in renderer, read from a `window.__DESKTOP_ENV__` object injected by preload, OR just skip the URL construction and show a "Preview generated" message with specs (format, bitrate, duration) as a text summary instead. This is safer — no env var threading needed in renderer.
7. Styling: follow LobbyScreen visual conventions exactly (#0d0d0d bg, #e8dcc8 text, #c8914a accent, #2a2218 borders).
8. Back/cancel button calls `onCancel()`.
