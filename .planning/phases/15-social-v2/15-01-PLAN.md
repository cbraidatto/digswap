## Objective

Build the data layer for trade-scoped messaging: schema, RLS, query helpers, and server actions. No UI in this plan — that's 15-02.

## Context

`trade_requests` has `requester_id` and `provider_id` as participants. `trade_runtime_sessions` tracks desktop presence. Neither has messaging. This plan adds:

- `trade_messages` table for per-trade threads
- Read state embedded in `trade_requests` (2 columns, no join required for unread count)
- Server-side actions with auth + rate limit enforcement

## Tasks

### Task 1 — Migration: trade_messages + read state

File: `supabase/migrations/20260401_trade_messages.sql`

```sql
-- trade_messages
create table if not exists public.trade_messages (
  id          uuid primary key default gen_random_uuid(),
  trade_id    uuid not null references public.trade_requests(id) on delete cascade,
  sender_id   uuid not null references auth.users(id) on delete cascade,
  kind        varchar(16) not null default 'user',
  body        text not null,
  created_at  timestamptz not null default timezone('utc', now()),
  constraint trade_messages_kind_check check (kind in ('user', 'system')),
  constraint trade_messages_body_length check (char_length(body) between 1 and 2000)
);

create index if not exists trade_messages_trade_id_created_at_idx
  on public.trade_messages (trade_id, created_at);

-- Read state on trade_requests
alter table public.trade_requests
  add column if not exists requester_last_read_at timestamptz,
  add column if not exists provider_last_read_at  timestamptz;

-- RLS
alter table public.trade_messages enable row level security;

-- Participants can read messages for their trades
create policy trade_messages_select_participants
  on public.trade_messages for select
  to authenticated
  using (
    exists (
      select 1 from public.trade_requests tr
      where tr.id = trade_id
        and (tr.requester_id = auth.uid() or tr.provider_id = auth.uid())
    )
  );

-- Participants can insert user messages for their own trades
-- system messages are service_role only (no authenticated insert policy for kind='system')
create policy trade_messages_insert_participants
  on public.trade_messages for insert
  to authenticated
  with check (
    sender_id = auth.uid()
    and kind = 'user'
    and exists (
      select 1 from public.trade_requests tr
      where tr.id = trade_id
        and (tr.requester_id = auth.uid() or tr.provider_id = auth.uid())
    )
  );
```
