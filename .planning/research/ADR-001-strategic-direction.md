# ADR-001 — DigSwap: Strategic Direction

**Status:** Approved
**Date:** 2026-03-28
**Participants:** Claude, Codex, Gemini
**Context:** Product at Phase 9 complete (45/48 plans). P2P functional. Remaining phases: 10, 11, 12.

---

## 1. Problem This Document Solves

The product was built as "a social network for vinyl collectors with P2P audio trading." That narrative has three failures:

1. **"Social network"** is too broad to move a serious digger
2. **"P2P audio trading"** as a headline attracts the wrong audience and repels influencers, partners, and press
3. The product lacked a **third mode of use** — not passive (notifications) nor discovery (search), but **active hunting work**

---

## 2. Decisions

### 2.1 Positioning

**What we are:**
> DigSwap is where serious diggers find who has the record they're hunting, build trust with strangers on the internet, and organize the hunt.

**External category:** "The discovery layer serious diggers have been missing"

**What we do not say:**
- "social network for vinyl collectors"
- "audio rip trading"
- "gamified record app"

---

### 2.2 The Three Product Pillars

| Pillar | Name | Delivers |
|--------|------|----------|
| 1 | **Discover** | The Radar, wantlist matching, alerts, rarity scores |
| 2 | **Connect** | Profiles, trust, P2P trades |
| 3 | **Work** | Digger Memory, contextual crate browsing, pressing research |

Pillar 3 is what transforms the app from "notification you check sometimes" to "tool you open when you go digging."

---

### 2.3 Onboarding Gate

**Decision:** Discogs sync completion required. No disc count minimum.

**Rationale:** The 100-disc barrier filters the organic distribution layer. The digger with 60 records is in contact with the one who has 800. Discogs sync already filters intent without killing word-of-mouth.

**Implementation rule:** Connect Discogs → start import → proceed to home with clear status. Never a dead end. Never fully blocking. But never skippable.

---

### 2.4 P2P

**Decision:** Never appears in marketing. Always present in the product.

**Non-negotiable implementation rule:** P2P is a mechanism, never a message. The first read of any surface involving P2P must be curation, collection, and context — never file inventory. This rule is transversal: applies to owners-list, request-audio-button, Crate Drop, and any new surface in phases 10–12.

---

### 2.5 Viral Acquisition Funnel

```
Rarity Score Card  →  pulls attention     (vanity + identity)
The Radar Receipt  →  sells utility       (magic moment)
Bounty Link        →  converts action     (bidirectional utility)
```

**Rarity Score Card**
Shareable image generated by the app. "Your collection is more obscure than 94% of users." Share button for Instagram Stories. Free on all plans.

**The Radar Receipt**
When the Radar fires, the result is shareable: "DigSwap found 3 people with records from my wantlist." Link goes to the user's public profile. Visitors see the collection and what's being hunted.

**Bounty Link**
User selects up to 3 Holy Grails and generates a public link: `digswap.com/u/username/bounty`. Share to bio, WhatsApp, anywhere. Visitors who have one of the records create an account to connect. Acquisition driven by real interest, not vanity.

**Crate Drop Link** *(with implementation rule)*
When a user marks rare records as available for trade, generates a "open my crate" link. Entry is curation — title, context, rarity scores. P2P appears only inside the experience as a request mechanism. Never as "files available for download."

**Strong public profile**
`digswap.com/u/username` becomes the serious collector's calling card. Richer than an Instagram @ because it has real collection data, trade reputation, and scene context. Every existing profile is an acquisition entry point without campaigns or seeding.

---

### 2.6 Digger Memory — MVP Specification

**Principle:** The app must remember the hunt better than WhatsApp and the Notes app. If context surfaces automatically before the user needs to dig for it, habit forms.

**Data structure:**
```typescript
leads {
  id, user_id,
  target_type: 'release' | 'user' | 'radar_match',
  target_id,
  note: text,
  status: 'watching' | 'contacted' | 'dead_end' | 'found',
  created_at, updated_at
}
RLS: select/insert/update by owner only
```

**Structural layer (Gemini):**
| Component | What it is |
|-----------|-----------|
| Lead | User, release, or Radar result marked as relevant |
| Note | Private free text linked to the lead |
| Status | watching / contacted / dead_end / found |
| My Hunts | Single screen with all leads, ordered by recent activity |

**UX layer (Codex):**
Notes and status appear as automatic tooltips anywhere in the app where the release or user appears. The Radar, search results, crate browsing — all surface saved context without the user needing to look for it.

**Schema must exist before Sprint 1.** The Radar in Sprint 1 already requires inline note + status on each match card.

---

### 2.7 What We Explicitly Do Not Build

| Feature | Reason |
|---------|--------|
| Token economy for P2P | Native reputation from Phase 9 already solves leeching |
| 100-disc barrier | Kills organic distribution |
| Kanban-style hunt boards | Unnecessary scope — lead + status + My Hunts resolves it |
| Full label intelligence | Phase 2+ when data density exists |
| Formal hunt sessions | Same reason |
| Real-time "currently online" | Infrastructure complexity without proportional v1 return |
| Physical vinyl marketplace/escrow | Discogs owns this — competing is suicide |

---

## 3. GTM — The One Thing a Document Cannot Solve

The product strategy is settled. The cold-start is not.

No viral mechanism works with an empty network. The only solution is personal onboarding of 30–50 serious collectors from one specific scene before any public launch. Not a generic scene — a concrete group with active wantlists and dense enough collections for the Radar to fire between them in the first days.

When the Radar fires inside that initial group, they talk. Not because the app asked — because finding a holy grail after years of hunting is the kind of thing you tell everyone.

Everything else depends on that moment happening first.
